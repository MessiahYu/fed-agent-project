# FedWatch AI V3 系统文档

> **版本**：V3  
> **目标**：在 V2 四路信号融合框架的基础上，通过**分层随机划分**消除时间固定效应，获得更无偏的模型泛化评估  
> **核心改进**：将 2000-2024 年历史数据的训练/测试划分方式，从"时间切割"改为"分层随机 80/20 抽样"

---

## 一、从 V2 到 V3 的改进

### 1.1 V2 划分方式的问题：时间固定效应

V2 采用时间切割：2000-2020 为训练集，2021-2024 为验证集。这种方式存在一个系统性偏差——**验证集集中在一个特殊的经济时期**（疫后高通胀 + 快速加息），与训练集覆盖的多元经济周期在结构上显著不同。

这种偏差称为**时间固定效应（Time Fixed Effects）**：同一年或邻近几年内，美联储的决策往往受到该时期特有的宏观背景影响（如 2001 年 9/11 + 反恐战争的宽松冲动、2008 年金融危机的零利率政策、2021-2023 年疫后通胀的激进加息），使相近年份的观测结果在统计上相互关联，而非独立同分布（i.i.d.）。

当训练集和验证集分别对应不同时代时，模型评估结果会受到以下两种扭曲：
1. **验证集基线异常低或高**：2021-2024 加息周期里 HIKE 样本多，基线（永远猜 HOLD）只有 56.2%，模型超越基线显得容易
2. **过拟合到某一周期模式**：模型可能只是学到了"加息期该如何判断"，而非普遍性的决策规律

### 1.2 V3 的解决方案：分层随机 80/20 抽样

**核心思路**：对 2000-2024 全部 206 次历史会议，按 HOLD / HIKE / CUT 三类决策类型分别独立随机抽取，80% 分入训练集，20% 分入测试集。这样：

- 训练集和测试集都**跨越所有经济周期**（互联网泡沫、金融危机、零利率时代、新冠冲击、加息周期等）
- 两个集合的**类别比例完全一致**（分层保证比例不偏）
- 验证出来的准确率代表模型在**任意时期**的泛化能力，而非特定周期

| 对比维度 | V2（时间切割）| V3（分层随机）|
|---------|-------------|-------------|
| 训练集范围 | 2000-2020（174 次）| 随机 80%（165 次，跨 2000-2024）|
| 测试集范围 | 2021-2024（32 次）| 随机 20%（41 次，跨 2000-2024）|
| 测试集 HOLD 基线 | 56.2%（加息期 HIKE 多）| **65.9%**（类别分布均衡）|
| 测试集 CUT 样本数 | 3 次（极少）| **6 次**（翻倍，更有统计意义）|
| 是否消除时间固定效应 | 否 | **是** |

## 二、设计逻辑与基本思想

V3 继承 V2 的全部信号设计和融合框架，**只改变了数据划分方式**。这使得两个版本的结果差异完全可归因于"划分方式"这一单一变量，具有干净的因果可解释性。

分层随机划分的理论依据来自**随机对照实验（RCT）**的逻辑：通过随机分配，使处理组（训练集）和对照组（测试集）在所有可观测和不可观测特征上分布一致，从而排除混淆因素（这里的混淆因素就是"所处时代"）。

---

## 三、分层随机划分详解

### 3.1 抽样流程

```
全部 206 次历史会议（2000-2024，split=train 或 test）
      ↓
按 actual_decision 分三组：
  HOLD 135 次  HIKE 40 次  CUT 31 次
      ↓
每组独立随机打乱（random seed = 42，保证可复现）
      ↓
  各组前 80% → 训练集 train
  各组后 20% → 测试集 test
      ↓
合计：train 165 次，test 41 次
（2025-至今 12 次 final_test 保持不变，不参与重新划分）
```

### 3.2 各类别分层抽样明细

| 类别 | 总数 | 训练集（80%）| 测试集（20%）|
|------|------|------------|------------|
| HOLD | 135 次 | 108 次 | 27 次 |
| HIKE | 40 次  | 32 次  | 8 次  |
| CUT  | 31 次  | 25 次  | 6 次  |
| **合计** | **206 次** | **165 次** | **41 次** |

### 3.3 可复现性保证

脚本 `v3/split_strategy.py` 使用 `numpy.random.default_rng(seed=42)` 固定随机种子，划分结果完全可复现。每次运行产生的 `v3/v3_split_assignments.csv` 是完整的审计日志，记录每次会议被分配到哪个集合。

## 四、最优参数（V3 网格搜索结果）

在 V3 训练集（165 次，跨 2000-2024 随机 80%）上遍历与 V2 完全相同的 6012 个参数组合，以训练集准确率最大化为目标。

### 4.1 V3 最优权重

| 参数 | V2 最优值 | V3 最优值 | 变化 | 含义 |
|------|---------|---------|------|------|
| `w_llm` | 0.45 | **0.35** | ↓ 0.10 | LLM 语义信号权重降低 |
| `w_var` | 0.20 | **0.35** | **↑ 0.15** | VAR 计量信号权重显著提升 |
| `w_market` | 0.15 | **0.10** | ↓ 0.05 | 市场信号权重小幅降低 |
| `w_pol` | 0.20 | **0.20** | → | 政治压力信号权重不变 |
| `hike_thr` | +0.25 | **+0.20** | ↓ 0.05 | 判断加息的门槛更低（更积极）|
| `cut_thr` | -0.35 | **-0.35** | → | 判断降息的门槛不变 |

### 4.2 权重变化的经济含义

**VAR 权重从 0.20 大幅升至 0.35** 是 V3 最显著的参数变化。

V2 的验证集集中在 2021-2024 加息周期，这段时期 Fed 的政策主要由通胀叙事（语言信号）驱动，导致 LLM 信号最有预测力，最优权重达到 0.45。

V3 的训练集和测试集覆盖所有周期，包含金融危机（宏观数据急剧恶化）、零利率时期（VAR 能准确捕捉利率通道效应）、疫情冲击（宏观数据领先于文本信号）等，这些场景中 VAR 的计量判断更可靠。因此，当训练数据覆盖更全面的经济周期时，算法自动给 VAR 分配了更高的权重。

这说明 **"LLM 优先"还是"VAR 优先"本身是一个数据驱动的结论，而非先验假设**——不同时代的数据结构会让最优权重漂移。

## 五、技术实现

V3 在代码层面复用 V2 的全部核心模块，只新增两个文件：

| 文件 | 作用 |
|------|------|
| `v3/split_strategy.py` | 分层随机划分脚本；读取数据库，重新写入 `split` 列；输出审计 CSV |
| `v3/final_test_eval_v3.py` | 最终测试集评估；从 config.yml 读取 V3 权重，复用 V2 的特征矩阵加载和 VAR 计算 |

复用的 V2 模块（无需修改）：
- `v2/weight_optimizer.py`：特征矩阵加载、VAR 信号计算、网格搜索框架
- `v2/backtest.py`：VAR 走一步预测、FRED 宏观数据拉取
- `utils/db.py`：数据库连接

### 6.2 V3 测试集各类别指标（41 次，跨 2000-2024）

| 类别 | 精确率 | 召回率 | F1 | 样本数 |
|------|--------|--------|-----|--------|

**关键发现**：HIKE 召回率达到 100%（8/8 全部识别），但 HOLD 召回率仅 55.6%（27 次中只猜对 15 次）。模型系统性地在高不确定性时期（如 2008-2009 金融危机、2009 年后量化宽松期）将 HOLD 误判为 CUT，因为政治压力信号（高 EPU、高 VIX）在这些时期极高，推低了融合分，触发 cut_thr。这是 V3 揭示的**真实模型弱点**——高不确定性不等于立即降息，模型混淆了"环境恶劣"和"行动迫切"。

**V3 vs V2 的两处预测差异：**

- **2025-01-29**：V3 融合分 -0.382（低于 cut_thr=-0.35），误判为 CUT；V2 融合分 -0.222 判为 HOLD（正确）。V3 的更高 VAR 权重（CUT 信号）把融合分拉过了阈值。

- **2026-01-28**：V3 融合分 -0.434，正确判断为 CUT；V2 融合分 -0.279 误判为 HOLD。这次 V3 因更高 VAR 权重反而抓住了实际降息。两处改变相互抵消，最终测试集准确率持平（均为 75%）。

### 6.4 最终测试集的核心突破：政策转变识别能力

**75% 的原始准确率存在误导性**。12 次会议中有 9 次是 HOLD，一个永远猜 HOLD 的"零智能基线"同样得到 75%。因此，原始准确率并不能说明模型的真实价值。

真正衡量模型能力的指标，是它能否在**政策转变的关键节点**发出正确信号。最终测试集共有 3 次非 HOLD 事件（全部为 CUT）：

| 会议日期 | 实际决策 | V3 预测 | 基线预测 | V3 是否识别转变 |
|---------|---------|--------|---------|--------------|
| 2025-10-29 | CUT | HOLD ✗ | HOLD ✗ | 未识别 |
| 2025-12-10 | CUT | **CUT ✓** | HOLD ✗ | **识别** |
| 2026-01-28 | CUT | **CUT ✓** | HOLD ✗ | **识别** |

**V3 在 3 次降息中成功识别了 2 次（67% 转变召回率），基线一次都无法捕捉（0%）。**

这正是该系统设计的核心价值所在：模型不是在重复"大概率维持不变"这个任何人都知道的结论，而是能在宏观信号积累到一定程度时，在市场和媒体普遍预期 HOLD 的背景下，**提前发出政策转向信号**。

对于债券投资和利率衍生品交易而言，HOLD 预测没有交易价值；能够判断"本次会议是否会偏离惯性"才是信息含量所在。从这个角度看，V3 在最近 2 年的关键决策点上捕捉到了 2/3 的转变，跑赢了基线的 0/3。

**为什么第 3 次（2025-10-29）没能识别？**

该次融合分为 -0.213，距 cut_thr=-0.35 仍差 0.137。LLM 分 -0.35（偏鸽），但 VAR 和市场信号均为 HOLD，两者拉升了融合分，未能突破阈值。事后看，这次降息在市场上也被认为是轻度意外——这说明模型的误判与市场的集体误判方向一致，并非模型特有的系统性错误。

## 七、局限性与改进方向

| 局限 | V3 新发现 | 可能的改进方向 |
|------|---------|-------------|
| HOLD 召回率低（55.6%）| 随机划分暴露了高不确定性 ≠ 立即行动的混淆 | 增加"利率已处于边界"特征（如零利率下界哑变量）|
| VAR 权重时变性 | 最优权重随覆盖周期不同而漂移（0.20→0.35）| 用滚动窗口重优化权重，而非固定全样本最优 |
| 测试集超越基线 +0.0pp | 基线更高（65.9%），模型改进空间更小 | 以 F1（macro-averaged）而非准确率为优化目标 |
| 新主席误判 | Warsh 首次会议预测为 HIKE，实际 HOLD | 增加"主席新任首次会议"哑变量，或专项蒸馏 |

## 八、运行方式

```bash
# 激活环境
conda activate fed-agent

# Step 1：执行分层随机划分（更新数据库 split 列）
python v3/split_strategy.py
# 加 --dry-run 只看结果不写库；加 --seed 123 更换随机种子

# Step 2：在新划分上重新运行权重优化
python v2/weight_optimizer.py
# 最优权重自动写入 config.yml → v3_optimal_weights

# Step 3：用 V3 权重评估最终测试集
python v3/final_test_eval_v3.py
```
