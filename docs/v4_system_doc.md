# FedWatch AI V4 系统文档

> **版本**：V4  
> **目标**：在 V2 四路信号融合框架基础上，通过 **LLM 信号滞后一期** 消除同期信息泄露，获得更真实的前瞻预测能力  
> **核心改进**：用上一次 FOMC 会议声明的鹰鸽分，预测本次会议的决策方向

---

## 一、从 V2/V3 到 V4 的改进

### 1.1 原有设计的根本性问题：同期信息泄露（Contemporaneous Information Leakage）

V2 和 V3 的特征矩阵在构建时，使用了**当次会议声明的鹰鸽分**（`hawkish_score`）来预测**当次会议的利率决策**。

这里存在一个被忽视的数据生成机制问题——**FOMC 声明原文和利率决策在同一时刻对外公布**：

> 每次 FOMC 会议结束后，美联储于美东时间下午 2:00 同步发布：  
> ① 利率决策公告（升/降/维持）  
> ② 会议声明全文（解释决策理由）

这意味着，声明原文与利率决策是**同一决策过程的两个输出**，而非前者预测后者。用当次声明来"预测"当次决策，在逻辑上是**同期回归（contemporaneous regression）**，不是真正的前瞻预测。

用一个通俗比喻：好比用一个人出门后的穿搭（声明）来"预测"他当天的出行计划（决策）——穿搭和出行计划本来就是同时决定的，这种"预测"在现实中毫无操作意义，因为你在决策公布之前根本拿不到声明原文。

### 1.2 V4 的修正方案：LLM 信号滞后一期

**核心思路**：将 `hawkish_score` 及其衍生字段（`inflation_concern`、`growth_concern`）整体**向后移动一期（shift(1)）**，使每次会议的 LLM 特征值等于**上一次会议**声明中提取的情绪分数。

```
V2/V3 的逻辑（存在泄露）：
  第 N 次会议声明（鹰鸽分）→ 预测第 N 次会议决策   ✗ 同期信息

V4 的修正逻辑：
  第 N-1 次会议声明（鹰鸽分）→ 预测第 N 次会议决策  ✓ 真正前瞻
```

经济直觉上，这也完全合理：**上一次声明中的措辞取向，是当前政策立场的有效"领先指标"**。FOMC 政策存在路径依赖（forward guidance），主席在上次声明中的措辞变化，往往预告了下次行动的方向。

### 1.3 为什么 V4 基于 V2 而非 V3？

V3 采用**分层随机 80/20 抽样**，将 2000-2024 年的会议打乱后随机分配到训练集和测试集。这导致：

- 测试集中的会议在时间上是**非连续的**（例如 2003 年某次会议紧挨着 2018 年的某次会议）
- 对这些非连续样本做 `shift(1)`，得到的"上一次"会议可能与当次相隔多年，**滞后特征失去经济含义**

V4 回归到 V2 的**时间切割划分**（2000-2020 训练，2021-2024 测试），在每个子集内部，会议按时间顺序连续排列，`shift(1)` 操作产生的是**真实的"相邻上一次"**，逻辑一致。

| 对比维度 | V3 方案（被排除）| V4 方案（采用）|
|---------|----------------|--------------|
| 数据划分 | 分层随机（消除时间固定效应）| 时间切割 V2 方式 |
| `shift(1)` 意义 | 可能跨越多年（无意义）| 相邻会议（有意义）|
| 同期泄露 | 存在 | **已消除** |
| 最适合场景 | 学术无偏估计 | **真实预测能力评估** |

## 二、设计逻辑与基本思想

V4 继承 V2 的全部信号框架（四路融合 + 网格搜索 + VAR），**只改变 LLM 特征的时间对齐方式**。

| 特征来源 | V2/V3 | V4 |
|---------|-------|-----|
| LLM 鹰鸽分 | 当次会议声明（同期）| **上次会议声明（滞后 1 期）**|
| VAR 计量信号 | 会议前 2 天截面数据（前瞻）| 同上（不变）|
| 市场信号 | 会议前 1 周数据（前瞻）| 同上（不变）|
| 政治压力信号 | 月度滚动百分位（前瞻）| 同上（不变）|

修改只在 `load_feature_matrix_v4()` 中的三行代码完成：

```python
for col in ["hawkish_score", "inflation_concern", "growth_concern"]:
    if col in df.columns:
        df[col] = df[col].shift(1)
df = df.dropna(subset=["hawkish_score"]).reset_index(drop=True)
```

`shift(1)` 后，序列中第一条记录（无上一次可参照）会产生 `NaN`，直接 `dropna` 丢弃，训练集从 173 次减至 172 次（减少了 2000 年第一次会议的那条记录）。

## 四、最优参数（V4 网格搜索结果）

在 V4 训练集（172 次，2000-2020，LLM 滞后一期）上遍历与 V2/V3 完全相同的 6012 个参数组合。

### 4.1 V4 最优权重

| 参数 | V2 最优值 | V3 最优值 | V4 最优值 | V4 变化说明 |
|------|---------|---------|---------|-----------|
| `w_llm` | 0.45 | 0.35 | **0.45** | 与 V2 相同；LLM 信号经滞后更干净，仍是最重要信号 |
| `w_var` | 0.20 | 0.35 | **0.30** | 介于 V2/V3 之间；VAR 重要性提升但不如 V3 极端 |
| `w_market` | 0.15 | 0.10 | **0.15** | 与 V2 相同 |
| `w_pol` | 0.20 | 0.20 | **0.10** | 政治压力信号权重降至最低；滞后 LLM 已隐含政策立场 |
| `hike_thr` | +0.25 | +0.20 | **+0.35** | 加息阈值提高；滞后 LLM 对 HIKE 的区分度更保守 |
| `cut_thr` | -0.35 | -0.35 | **-0.35** | 降息阈值不变 |

### 4.2 权重变化的经济含义

最显著的变化是**政治压力权重从 0.20 降至 0.10**。

原因在于：V2/V3 中，当次声明分数本质上还"含有"当次利率决策的信息（同期泄露），模型需要借助政治压力信号（EPU/VIX）来增加多样性。而 V4 的 LLM 分来自上一次声明，已经是一个干净的前瞻信号，且其中已隐含了上一次会议时的政策氛围；这使得政治压力信号对预测的边际贡献减少，网格搜索自然降低了它的权重。

**加息阈值从 +0.25 提高到 +0.35** 的原因：滞后的 LLM 分与当次 HIKE 决策之间多了一个时间维度的衰减，同等水平的鹰派信号在一期后驱动 HIKE 的概率更低，需要更高的阈值才能触发加息预测，减少假阳性。

## 五、技术实现

V4 在代码层面复用 V2 的全部核心模块，只新增两个文件：

| 文件 | 作用 |
|------|------|
| `v4/weight_optimizer_v4.py` | 在 V2 权重优化器基础上，重写 `load_feature_matrix_v4()` 做 shift(1)；重新跑 6012 组网格搜索；最优权重写入 config.yml |
| `v4/final_test_eval_v4.py` | 用 V4 特征矩阵（含滞后 LLM）评估最终 12 次会议；与 "V2 权重 + V4 特征" 对比验证权重影响 |

**核心代码片段**（`load_feature_matrix_v4()` 的关键修改）：

```python
# 重新应用 V2 时间切割划分（覆盖数据库中 V3 随机划分的标签）
df["split"] = "train"
df.loc[df["meeting_date"] >= "2021-01-01", "split"] = "test"
df.loc[df["meeting_date"] >= "2025-01-01", "split"] = "final_test"

# V4 核心改动：LLM 信号滞后一期
for col in ["hawkish_score", "inflation_concern", "growth_concern"]:
    if col in df.columns:
        df[col] = df[col].shift(1)
df = df.dropna(subset=["hawkish_score"]).reset_index(drop=True)
```

### 6.3 关键修正详解：沃什首次会议

这是 V4 最重要的预测修正案例。

**V2/V3 的错误路径**（同期特征）：
```
2026-06-17 沃什声明：
"Inflation remains elevated"、"price stability is paramount"
→ LLM 分：+0.15（当次声明，偏鹰）
→ 融合分超过 hike_thr=+0.25
→ 预测 HIKE   ✗（实际 HOLD）
```

**V4 的正确路径**（滞后特征）：
```
2026-04-29 鲍威尔声明（V4 使用这个）：
→ LLM 分：-0.25（上次声明，偏鸽）
→ 融合分 = 0.45×(-0.25) + 0.30×0 + 0.15×0 + 0.10×(pol) ≈ -0.012
→ 预测 HOLD   ✓
```

**背后的经济逻辑**：沃什 2026-06-17 上任后的首次会议，他的声明确实措辞偏鹰——但这是他作为新主席建立信誉时的**定调**，而非行动信号。全票维持利率不变，沃什也明确表示"有降息空间"。V2/V3 被声明的鹰派措辞所误导；V4 正确地使用了"本次决策之前能看到的最新信号"（鲍威尔上次声明），避免了同期泄露。

### 6.4 政策转变识别能力

V4 在 3 次 CUT 事件上的表现：

| 会议日期 | 实际 | V4 预测 | 基线 | V2（原）| V3 | V4 识别？ |
|---------|------|--------|------|--------|-----|---------|
| 2025-10-29 | CUT | HOLD ✗ | HOLD ✗ | HOLD ✗ | HOLD ✗ | 未识别 |
| 2025-12-10 | CUT | **CUT ✓** | HOLD ✗ | CUT ✓ | CUT ✓ | **识别** |
| 2026-01-28 | CUT | **CUT ✓** | HOLD ✗ | HOLD ✗ | CUT ✓ | **识别** |

**V4 CUT 召回率：66.7%（2/3）**，与 V3 持平；但 V4 同时修正了沃什会议的 HIKE 误判，总准确率从 75% 提升至 91.7%。

**V4 为什么仍然错过 2025-10-29 CUT？**

该次 LLM 分为 -0.55（来自 9 月会议声明，非常鸽派），但 VAR 和市场信号均为 HOLD；融合分 = 0.45×(-0.55) + 0.30×0 + ... = -0.293，距 cut_thr=-0.35 差距 0.057。

事后分析：2025-10-29 的降息在市场上也被视为轻度意外。这说明该次决策的信号相对模糊，模型与市场共同低估了降息概率，并非模型特有的系统性偏差。

## 八、局限性与改进方向

| 局限 | V4 表现 | 可能的改进方向 |
|------|--------|-------------|
| 2025-10-29 CUT 漏判 | 融合分 -0.293，离 cut_thr=-0.35 差 0.057 | 动态调整 cut_thr（如基于过去 N 次会议的置信区间）|
| 滞后信号的"信息衰减" | 上次声明与本次决策间经济形势可能骤变 | 引入 LLM 分的"变化量"（Δhawkish）作为额外特征 |
| 加息阈值偏高（+0.35）| 对 HIKE 的预测更保守，可能错过快速加息 | 在通胀冲击期用更低的 hike_thr，分情景设置阈值 |
| 历史样本 172 次 | 仍属中等规模，参数估计有不确定性 | 扩展至 1950 年代以来全量 FOMC 数据 |
