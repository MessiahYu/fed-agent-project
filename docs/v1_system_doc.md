# FedWatch AI V1 系统文档

> **版本**：V1  
> **目标**：实时预测美联储 FOMC 议息会议政策走向（加息 / 降息 / 维持不变）  
> **核心框架**：CrewAI 多智能体编排 + DeepSeek LLM + FRED 宏观数据  

---

## 一、设计逻辑

### 1.1 基本思想

美联储的货币政策决策由三类信息共同驱动：

| 信息类型 | 具体内容 | V1 的处理方式 |
|---------|---------|-------------|
| **文本信号** | FOMC 声明原文的措辞与语气 | LLM 语义分析 |
| **计量信号** | CPI、失业率、联邦基金利率的时序规律 | VAR 向量自回归模型 |
| **外部压力** | 市场预期、政治压力、国际货币政策联动 | 定量数据 + LLM 综合分析 |

V1 的核心主张是：**单一模型无法捕捉央行决策的全貌**。传统计量模型（VAR/VECM）只能看到数字，看不懂措辞；纯 LLM 能读文字，但缺乏对历史周期规律的量化校准。V1 的解法是将两者拆解为独立智能体，分别给出信号，再由决策智能体综合判断。

### 1.2 为什么用多智能体而不是单一模型

- **职责清晰**：每个智能体只做一件事，输出标准化结构体，便于追溯哪个环节出现了偏差
- **可解释性**：最终结论附带"证据链"（evidence_chain）和"推理步骤"（chain-of-draft），可以逐句追溯为什么这么判断
- **内生性问题的规避**：美联储声明的语气本身会影响市场，市场反应又会影响下次声明——传统计量因果识别困难。多智能体架构把语义信号和计量信号分离处理，避免两者在单一回归中互相混淆

## 二、系统架构：五智能体流水线

```
[美联储官网]  [FRED API]
      ↓            ↓
  Agent 1：数据感知智能体
      ↓
  Agent 2：语义特征提取智能体
      ↓              ↘
  Agent 3：决策推演智能体  ←  Agent 4：主席行为蒸馏智能体
                              Agent 5：影响者联合智能体
      ↓
  最终预测（结构化 JSON）
```

---

## 三、各智能体详解

### Agent 1 —— 数据感知智能体（Data Perception）

**职责**：作为系统的"眼睛"，实时抓取所有原始数据，打包成统一格式传给下游。

**数据来源**：

| 数据 | 来源 | 内容 |
|------|------|------|
| FOMC 声明原文 | federalreserve.gov | 最新一次会议发布的政策声明全文 |
| CPI 同比增速 | FRED `CPIAUCSL` | 消费者价格指数，衡量通胀 |
| 失业率 | FRED `UNRATE` | 衡量就业市场冷热 |
| 联邦基金利率 | FRED `FEDFUNDS` | 当前利率基准水平 |
| 10 年期国债收益率 | FRED `DGS10` | 长期利率预期锚 |

**关键设计：防前视偏差（Look-Ahead Bias Prevention）**  
系统在取宏观数据时，截断点设为**会议日前两天**，只取那时能公开获取的数据，不使用会议后才发布的数据。这模拟了真实分析师在开会前的信息集合。

**输出结构**（`MeetingPackage`）：
```
meeting_date     会议日期
statement_url    声明来源 URL
statement_text   声明正文（已清洗）
macro            宏观指标快照
data_cutoff_date 数据截断日期
source           live（实时）或 fallback（本地备用）
```

---

### Agent 2 —— 语义特征提取智能体（Semantic Extraction）

**职责**：把声明的自然语言转化为可计算的数值特征，是系统的"翻译官"。

**工作原理**：  
以资深美联储分析师的角色提示 DeepSeek LLM，要求逐句分析声明中的关键措辞变化（如 `"somewhat elevated"` vs `"elevated"` 代表不同程度的通胀警惕性），结合宏观背景校准评分，最终输出结构化特征。

**输出结构**（`SemanticFeature`）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `hawkish_score` | float，-1.0 ~ +1.0 | 鹰鸽评分。+1 = 极鹰（倾向加息），-1 = 极鸽（倾向降息），0 = 中性 |
| `inflation_concern` | float，0.0 ~ 1.0 | 对通胀的担忧程度 |
| `growth_concern` | float，0.0 ~ 1.0 | 对经济增长的担忧程度 |
| `key_evidence` | list[str] | 3-5 条关键证据，直接引用声明原文 |
| `warsh_signal` | str | 对现任主席沃什立场的判断：hawkish / neutral / dovish |
| `conclusion` | str | 政策走向结论，100 字以内 |

---

### Agent 3 —— 决策推演智能体（Decision Inference）

**职责**：综合所有上游信号，给出对下次 FOMC 会议的最终预测，是系统的"大脑"。

**两路信号融合**：

**① VAR 计量基线**（纯数据，不调 LLM）  
- 从 FRED 拉取近 5 年月度历史数据：联邦基金利率、CPI 同比、失业率  
- 拟合向量自回归模型（VAR），预测下期利率方向  
- 作用：提供历史数据驱动的计量基准，锚定 LLM 不过度发散  

**② LLM Chain-of-Draft 推理**  
- 把 Agent 2 的语义特征、Agent 4 的主席画像、Agent 5 的外部压力综合输入 LLM  
- 采用 Chain-of-Draft 分五步推理：通胀评估 → 就业评估 → 主席风格 → 外部压力 → 最终决策  
- 作用：处理计量模型捕捉不到的非线性判断（如地缘政治冲击、主席个人风格）  

**输出结构**（`FOMCPrediction`）：

| 字段 | 说明 |
|------|------|
| `predicted_action` | 预测决策：hold / cut_25bp / cut_50bp / hike_25bp |
| `confidence` | 综合置信度，0.0 ~ 1.0 |
| `var_signal` | VAR 模型信号：hold / cut / hike |
| `llm_signal` | LLM 推演信号：hold / cut / hike |
| `cod_steps` | Chain-of-Draft 五步推理过程 |
| `evidence_chain` | 支撑预测的证据链 |
| `risk_factors` | 关键风险因素（可能推翻预测的情形）|
| `final_conclusion` | 综合结论，150 字以内 |

---

### Agent 4 —— 主席行为蒸馏智能体（Chair Distillation）

**职责**：分析历史各任主席的决策风格，推断现任主席沃什（Kevin Warsh）在当前宏观环境下的行动倾向。

**工作原理**：
1. 从 `fomc_history` 数据库中自动统计 4 位历史主席的量化决策画像  
   （例如：伯南克平均在 CPI 同比几何时降息？耶伦加息时失业率通常在什么水平？）  
2. 结合每位主席的定性画像（货币哲学、决策风格、历史评价）  
3. 让 LLM 以每位主席的视角分别"投票"  
4. 分析沃什最接近哪位主席的哪个历史时期，输出蒸馏后的预测  

**四位主席画像**：

| 主席 | 任期 | 核心风格 |
|------|------|---------|
| 格林斯潘 | 1987-2006 | 数据依赖，灵活务实，擅长预防性降息 |
| 伯南克 | 2006-2014 | 学术型，发明量化宽松，极度鸽派 |
| 耶伦 | 2014-2018 | 渐进主义，重就业，鸽中带鹰 |
| 鲍威尔 | 2018-2026 | 务实型，沟通透明，2022 年激进转鹰 |

**输出结构**（`DistillResult`）：各主席投票结果 + 理由 + 沃什最接近的历史时期 + 蒸馏后预测

---

### Agent 5 —— 影响者联合智能体（Influencer Network）

**职责**：捕捉美联储决策中来自市场、政治、国际三个方向的外部压力，是系统的"雷达"。

**三路子信号**：

**A. 市场预期（纯定量，不调 LLM）**  
- 数据：FRED `DGS2`（2 年期）、`DGS10`（10 年期）国债收益率  
- 指标：收益率曲线斜率 = DGS10 - DGS2  
  - 斜率正且扩大 → 市场预期加息  
  - 斜率负（倒挂）→ 市场预期降息或经济衰退  
- 逻辑：国债市场是全球流动性最好的市场，收益率曲线是集体预期的客观体现  

**B. 政治压力（LLM 定性分析）**  
- 输入：config.yml 中的政治背景描述（白宫/国会对美联储的施压方向）  
- LLM 分析政治压力的方向（鸽派压力 / 鹰派压力 / 中性）及对独立性的威胁程度  

**C. 国际联动（FRED 数据 + LLM）**  
- 数据：ECB 利率（`ECBDFR`）、日本央行利率（`IRSTJP01M156N`）、英国央行利率（`IUDSOIA`）、美元指数（`DTWEXBGS`）  
- LLM 分析主要央行政策方向与美联储的协同或背离，以及美元走势对美联储决策空间的约束  

**输出结构**（`InfluencerResult`）：各子信号评分 + 综合影响方向

| 类别 | 数据 | 来源 | 频率 |
|------|------|------|------|
| 美联储声明 | FOMC 政策声明原文 | federalreserve.gov | 每次会议（年 8 次）|

| 历史决策 | 2000-2024 年 FOMC 记录 | SQLite（fomc_history） | 静态 |

## 五、技术栈

| 组件 | 技术选型 | 作用 |
|------|---------|------|
| 智能体编排 | CrewAI | 定义智能体角色、任务、协作流程 |
| 大语言模型 | DeepSeek（deepseek-chat） | 语义理解、推理、结构化输出 |
| 计量模型 | statsmodels VAR | 宏观时序预测 |
| 数据结构化 | Pydantic | 所有输出强制类型校验，防止 LLM 乱输 |

| 数据接口 | FRED API + requests | 实时拉取宏观数据 |
| 配置管理 | config.yml + python-dotenv | 参数与密钥分离，所有可变值不硬编码 |

## 六、系统输出与预测能力

### 6.1 直接输出

运行 `v1/pipeline.py` 后，系统输出一份完整的预测报告，包含：

- **决策预测**：hold / cut_25bp / cut_50bp / hike_25bp
- **置信度**：0.0 ~ 1.0 的数值
- **鹰鸽评分**：-1.0 ~ +1.0（正 = 鹰 = 倾向加息）
- **五步推理过程**：从通胀分析到最终决策的完整逻辑链
- **关键证据**：3-5 条直接引用声明原文的依据
- **风险因素**：列举可能推翻预测的情景
- **交易信号**（示例）：做空国债 / 做多美元

### 6.2 V1 的定位

V1 是一个**实时单次预测系统**：给定当前最新声明和宏观数据，预测下一次 FOMC 会议的决策方向。它不做历史回测，也没有系统性的准确率统计。其优势在于：

- **实时性**：数据触发即可运行，无需等待人工整理
- **可解释性**：每一步推理都有完整记录，支持"逐条对话"式审查
- **灵活性**：可以针对沃什这样的新主席快速更新画像，不依赖大量历史数据标定

## 七、关键工程约束

### 7.1 配置与代码解耦

所有可变值（URL、模型名、阈值、长文本提示）均存储在 `config.yml`，`.py` 文件只包含逻辑。这样的好处：更换模型只改一行配置；调整提示词不需要触碰代码。

### 7.2 密钥安全

API 密钥（`DEEPSEEK_API_KEY`、`FRED_API_KEY`）仅存于 `.env` 文件，`.gitignore` 明确排除，任何情况下不打印、不提交。

### 7.3 降级处理

Agent 1 包含本地备用声明（`FOMC_FALLBACK_TEXT`），当美联储官网无法访问时自动切换，保证系统不因网络问题中断。

## 八、局限性

| 局限 | 原因 | 在 V2 中的改进方向 |
|------|------|-----------------|
| 无历史准确率统计 | V1 只做单次实时预测，没有回测框架 | V2 构建了 2000-2024 年完整历史数据库，支持系统性回测 |
| LLM 输出不稳定 | 同一段声明两次运行可能得出略有差异的 hawkish_score | V2 对 206 次历史会议批量分析，用统计平均平滑了单次波动 |
| 权重依赖经验 | Agent 3 融合信号的权重是人工设定的 | V2 通过网格搜索在 2000-2020 训练数据上自动优化权重 |
| 新主席适应期 | 沃什仅有 1 次正式决策记录，行为蒸馏样本量不足 | 随会议积累，Agent 4 的量化画像会自动更新 |

## 九、运行方式

```bash
# 激活环境
conda activate fed-agent

# 运行完整五智能体流水线
python v1/pipeline.py

# 或单独测试某个智能体
python v1/agent2_semantic.py
```

**依赖密钥**（放入项目根目录 `.env`）：
```
DEEPSEEK_API_KEY=your_key_here
FRED_API_KEY=your_key_here
```
