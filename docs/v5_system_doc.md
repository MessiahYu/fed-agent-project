# FedWatch AI V5 系统文档

> **版本**：V5  
> **目标**：在 V4 的基础上，同时引入 V3 的分层随机划分，一次性消除**同期信息泄露**和**时间固定效应**两个问题，获得方法论最完整的预测框架  
> **核心改进**：先按时间顺序计算 LLM 滞后特征，再做分层随机 80/20 划分——两步顺序不可颠倒

---

## 一、从 V3 + V4 到 V5 的改进

### 1.1 V3 和 V4 各自修复了一半

| 版本 | 修复了什么 | 还存在什么问题 |
|------|---------|-------------|
| V3 | 时间固定效应（分层随机划分，测试集跨越所有经济周期）| 同期信息泄露（仍用当次声明预测当次决策）|
| V4 | 同期信息泄露（LLM 信号滞后一期，用上次声明预测本次）| 时间固定效应（V2 时间切割，历史测试集集中在加息周期）|

V3 和 V4 是**互补的两个方向**，自然的问题是：能不能把两者合在一起？

### 1.2 直接叠加为什么行不通

看似可以"先做 V3 随机划分，再对 LLM 分做 shift(1)"，但这个顺序会破坏 lag 的经济含义：

```
❌ 错误顺序（先随机打乱再做 shift(1)）：

  打乱后数据顺序（举例）：
    第1行：2003-03 会议，hawkish_score = +0.20
    第2行：2018-09 会议，hawkish_score = +0.60
    第3行：2007-06 会议，hawkish_score = -0.10

  shift(1) 后：
    第1行：NaN（无前驱）
    第2行：+0.20（来自 2003-03，与 2018-09 相差 15 年）← 毫无经济意义
    第3行：+0.60（来自 2018-09，与 2007-06 倒序 11 年）← 毫无经济意义
```

shift(1) 的语义是"上一行"，而非"时间上的上一次"。随机打乱之后，"上一行"可能来自任何年份，完全失去了"上次会议声明"的经济含义。

### 1.3 V5 的正确做法：先锁定 lag，再随机划分

**关键洞察**：LLM 滞后特征是**特征工程步骤**，必须在时间顺序上完成，然后把结果"锁定"在每一条记录里。一旦每条记录的 lag 值被算好，后续的随机划分只影响哪些记录进训练集、哪些进测试集，不会改变 lag 的数值。

```
✓ 正确顺序（先锁定 lag，再随机划分）：

Step 1：按时间顺序加载全部 217 次会议
Step 2：在时间顺序上计算 shift(1)
  → 2003-03 的 lag = 2003-01 会议的 hawkish_score（相邻的真实前驱）
  → 2018-09 的 lag = 2018-08 会议的 hawkish_score（相邻的真实前驱）
  → lag 值已锁定在每条记录中，不会再变
Step 3：丢弃第一行（无前驱，lag = NaN）
Step 4：对历史会议（2000-2024）做分层随机 80/20 划分
  → 2003-03 可能进训练集，2018-09 可能进测试集
  → 但两者的 lag 值已经在 Step 2 算好，划分不影响它们

结果：
  每条记录的 lag 特征 = 时间上真实的前一次会议声明分数（V4 的修复保留）
  训练/测试集分布跨越所有经济周期（V3 的修复保留）
```

## 二、设计逻辑

V5 在代码层面只有两处核心改动（相对于 V4）：

1. **不使用 V2 时间切割**，改为分层随机 80/20 划分
2. **shift(1) 的位置不变**（仍在加载后按时间排序立即执行），划分逻辑在其之后

其余一切沿用 V2/V3/V4 的框架：四路信号、网格搜索 6012 组参数、走一步 VAR、FRED 数据。

```
V5 特征矩阵构建流程：

  load_feature_matrix()        ← 从数据库加载，含 hawkish_score（当期）
         ↓
  sort_values("meeting_ts")    ← 按时间排序（关键前置步骤）
         ↓
  df[LLM列] = shift(1)         ← LLM 特征整体后移一期（V4 修复，此时顺序正确）
  dropna()                     ← 丢弃第一行（无前驱）
         ↓
  分离 final_test（2025年以后） ← 不参与任何划分或优化
         ↓
  对历史会议做分层随机 80/20   ← V3 修复（对已含 lag 的记录随机划分）
         ↓
  合并：train + test + final_test
```

## 四、最优参数（V5 网格搜索结果）

在 V5 训练集（162次，2000-2024 随机 80%，含 LLM lag）上遍历 6012 个参数组合。

### 4.1 V5 最优权重

| 参数 | V3 | V4 | **V5** | V5 变化说明 |
|------|----|----|--------|-----------|
| `w_llm` | 0.35 | 0.45 | **0.40** | 介于 V3/V4 之间；lag 特征干净，但多周期中 VAR 同样重要 |
| `w_var` | 0.35 | 0.30 | **0.35** | 与 V3 相同；随机划分覆盖多周期，VAR 跨周期价值得到验证 |
| `w_market` | 0.10 | 0.15 | **0.10** | 与 V3 相同；市场信号边际贡献较小 |
| `w_pol` | 0.20 | 0.10 | **0.15** | 介于 V3/V4 之间 |
| `hike_thr` | +0.20 | +0.35 | **+0.35** | 与 V4 相同；滞后信号需要更高阈值才触发加息预测 |
| `cut_thr` | -0.35 | -0.35 | **-0.40** | 比 V4 更严格；随机划分下误判率更高，阈值收紧 |

### 4.2 权重变化的经济含义

**w_var 从 V4 的 0.30 回升到 0.35**：V4 使用 2021-2024 加息期作为测试集，该时期宏观信号非常清晰，VAR 很有价值。但因测试集结构特殊（信号清晰），V4 对 VAR 的权重估计可能不是最优。V5 使用跨越多周期的测试集，宏观信号复杂度更高，数据自然给 VAR 分配了 0.35 的权重——这与 V3 一致，说明在全周期数据下 VAR 的贡献稳定在 0.35 左右。

**cut_thr 从 -0.35 收紧到 -0.40**：V5 历史测试集包含金融危机、量化宽松期等高不确定性场景，这些时期政治压力信号（EPU/VIX）高企，容易把融合分推向负值但实际不降息——更严格的阈值减少了这类假阳性。代价是 2025-10-29 那次降息更难触发（融合分 -0.288，距阈值 -0.40 差 0.112）。

## 五、技术实现

V5 复用 V2/V3/V4 的全部核心模块，只新增两个文件：

| 文件 | 作用 |
|------|------|
| `v5/weight_optimizer_v5.py` | 内含 `load_feature_matrix_v5()`：先 shift(1)，再分层随机 80/20；跑 6012 组网格搜索；写入 config.yml |
| `v5/final_test_eval_v5.py` | 加载 V5 特征矩阵，评估最终 12 次会议，与 V4 对比 |

**核心代码（`load_feature_matrix_v5()` 关键逻辑）**：

```python
# Step 1: 按时间排序后立即计算 lag（顺序不可颠倒）
df = df.sort_values("meeting_ts").reset_index(drop=True)
for col in ["hawkish_score", "inflation_concern", "growth_concern"]:
    df[col] = df[col].shift(1)        # 时间顺序上的滞后一期
df = df.dropna(subset=["hawkish_score"])  # 丢弃首行

# Step 2: 分离 final_test（已含正确 lag 值）
final_mask = df["meeting_date"] >= "2025-01-01"
df_final = df[final_mask].copy()
df_hist  = df[~final_mask].copy()

# Step 3: 对历史会议做分层随机 80/20（不写库，只加标签）
df_hist = _stratified_split(df_hist)   # seed=42，可复现

# Step 4: 合并
df_final["split"] = "final_test"
return pd.concat([df_hist, df_final]).sort_values("meeting_ts")
```

### 6.4 V5 验证了 V4 的核心发现是稳健的

V5 和 V4 的最大区别是**用什么数据优化权重**：
- V4：用 2021-2024 加息周期（时代偏差，"容易"的测试集）
- V5：用 2000-2024 随机 20%（无偏，各种经济周期）

两者权重略有不同（V4：w_llm=0.45，V5：w_llm=0.40），但最终 12 次预测完全相同。

这说明：**LLM 信号滞后一期（shift(1)）才是驱动最终测试集从 75% 跃升至 91.7% 的根本原因，与权重具体取值无关。** 特征工程（用上次声明预测本次决策）是决定性改进，权重的精确数值是次要因素。

## 七、V5 暴露的新局限

| 局限 | V5 新发现 | 改进方向 |
|------|---------|---------|
| HIKE 召回率低（37.5%）| 随机划分把低加息期（量化宽松时代）纳入测试集，模型在这些场景下把 HIKE 判成 HOLD | 增加"利率绝对水平"特征（如是否处于零下界），帮助模型区分"没有 HIKE 压力"和"HIKE 信号弱" |
| cut_thr 收紧加大漏判 | -0.40 使 2025-10-29 差距从 0.057 扩大到 0.112 | 以 Macro F1 为优化目标而非 Accuracy，迫使模型在 CUT 误判与漏判之间取更好的平衡 |
| LLM 权重依然最高（0.40）| 说明滞后声明的语义信息仍是最强信号 | 引入 Δhawkish（相邻两次声明分的变化量），捕捉"趋势加速"而非只看绝对值 |
| 主席风格未建模 | 沃什首次会议在 V4/V5 碰巧因 lag 机制得到正确结果，但本质问题未解决 | 增加"新任主席首次会议"哑变量，或将主席历史行为画像作为第五路信号 |
