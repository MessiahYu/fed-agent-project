# FedWatch AI：基于多智能体编排的美联储政策预测系统

> **作者**：余青锋\
> **提交日期**：2026-06-21\
> **代码仓库**：<https://github.com/MessiahYu/fed-agent-project>

------------------------------------------------------------------------

## 一、项目概述

### 1.1 项目目标

本项目搭建一个**基于多智能体编排的持续性经济事件预测系统**，预测美联储（Fed）议息会议的政策走向（加息 / 降息 / 维持不变）。

核心思路是将四路异质信号（LLM 语义分 + VAR 计量模型 + 市场信号 + 政治压力信号）融合，通过网格搜索在历史数据上确定最优权重，再对一个**完全未参与训练的最终测试集（2025-01 至 2026-06，共 12 次 FOMC 会议）**进行独立评估，以此验证模型的泛化能力。

这 12 次会议横跨鲍威尔（Powell）主席晚期和沃什（Warsh）新任主席期，涵盖降息周期启动（2025 年底）与政策不确定性最高阶段（2026 年）。最终 V5 版本在 12 次中预测正确 11 次，准确率 **91.7%**，准确预测了三次降息决定中的两次。

在此基础上，**V6** 在 V5 量化融合框架之上引入真正的多智能体并行推理层：4 个信号 Agent 同时分析各自专业维度，由裁决 Agent 汇聚四路报告做出最终判断，并输出"风险归因"（哪路信号最决定性）。V6 在同一测试集上与 V5 准确率持平（91.7%），同时将黑箱数字转化为可解释的推理链，是系统最终的多智能体展示版本。

| 内生性 / 因果识别论证 | ✅ 走一步 VAR 严格防前视偏差，各信号独立构建避免联立内生性 |

### 1.3 多智能体框架：CrewAI

本系统基于 **CrewAI** 框架搭建多智能体编排，而非 LangChain 或 AutoGen。

选择 CrewAI 的理由： - 原生支持 **Role-Goal-Backstory** 角色设定，每个智能体有明确分工和行为约束 - 内置任务依赖链（Task → Crew），天然适合"数据→分析→决策"的流水线结构 - 支持结构化输出（output_pydantic），保证信号数值可直接参与下游计算

```
V1 / V6 CrewAI 编排结构：

  LLM（DeepSeek-Chat）
         ↓
  Agent（角色 / 目标 / 背景）
         ↓
  Task（expected_output 约束格式）
         ↓
  Crew.kickoff()
         ↓
  结构化输出 → 下游计算或裁决 Agent
```

**V6 新增：并行执行模式**

```
  Agent A（语义）──┐  async_execution=True
  Agent B（VAR） ──┤  四路同时发出 LLM 请求
  Agent C（市场）──┤
  Agent D（政治）──┘
         ↓
  Agent E（裁决） context=[A,B,C,D]
         ↓
  最终预测（HIKE/HOLD/CUT）+ 风险归因
```

V2–V5 是**多模块**（函数流水线，无 Crew 调度）；V1 和 V6 是**真正的多 Agent**（CrewAI 调度、LLM 驱动角色分工）。

### 1.4 机器学习组件分类

| 模块 | ML 类别 | 具体方法 |
|------------------------|------------------------|------------------------|
| **权重优化（网格搜索）** | **有监督学习** | 以历史 FOMC 决策为标签，在训练集上穷举 6012 个参数组合，最大化分类准确率 |
| **VAR 走一步预测** | **有监督学习（时序）** | 向量自回归预测下期联邦基金利率方向，走一步无前视偏差 |
| **LLM 语义分析** | **零样本推理（迁移学习）** | DeepSeek 基于预训练知识，无需额外标注样本对 FOMC 声明进行鹰鸽打分 |
| **政治压力信号** | **无监督特征工程** | EPU / VIX 滚动百分位计算，不使用任何决策标签 |

## 二、最终版本 V5 模型详解

V5 是本项目方法论最完整的版本，同时修复了前序版本中存在的两个核心问题：**时间固定效应**（V3 修复）与**同期信息泄露**（V4 修复）。

### 2.1 核心预测公式

```         
四路信号融合：

fusion_score = 0.40 × LLM语义分（上次声明）
             + 0.35 × VAR计量信号
             + 0.10 × 市场信号
             + 0.15 × 政治压力信号

决策规则：
  fusion_score > +0.35  →  预测 HIKE（加息）
  fusion_score < -0.40  →  预测 CUT（降息）
  其余                  →  预测 HOLD（维持不变）
```

**注意**：LLM 语义分使用的是**上一次 FOMC 会议声明**提取的鹰鸽分，而非当次声明。原因详见第三章 V4 节。

### 2.2 各信号数据来源与计算逻辑

| 信号 | 数据来源 | 计算逻辑 | 时间对齐 |
|------------------|------------------|------------------|------------------|
| LLM 语义分 | 美联储官网 FOMC 声明原文 | DeepSeek 输出 -1.0（极鸽）到 +1.0（极鹰） | 使用**上次**声明分数（shift(1)） |
| VAR 计量信号 | FRED：FEDFUNDS / CPI / UNRATE | 走一步 VAR，截断到会议前 2 天 | 无前视偏差 |
| 市场信号 | FRED：DGS2 / DGS10 | 收益率曲线斜率（10Y-2Y）→ hike / hold / cut | 会议前数据 |
| 政治压力 | FRED：EPU 指数 / VIX | 滚动 36 月百分位加权，映射到 [-1, +1] | 月频当期值 |

### 2.5 政策转折点识别能力

75% 的原始准确率存在误导性——12 次会议中 9 次是 HOLD，"永远猜维持"的零智能基线同样是 75%。真正的模型价值在于能否识别政策转变节点（3 次 CUT 事件）：

| 降息事件       | 基线   | V2        | V3        | V4        | **V5**    |
|----------------|--------|-----------|-----------|-----------|-----------|
| 2025-10-29 CUT | HOLD ✗ | HOLD ✗    | HOLD ✗    | HOLD ✗    | HOLD ✗    |
| 2025-12-10 CUT | HOLD ✗ | **CUT ✓** | **CUT ✓** | **CUT ✓** | **CUT ✓** |
| 2026-01-28 CUT | HOLD ✗ | HOLD ✗    | **CUT ✓** | **CUT ✓** | **CUT ✓** |
| **CUT 召回率** | **0%** | **33%**   | **67%**   | **67%**   | **67%**   |

从 V2 的 33% 到 V3/V4/V5 的 67%，系统在 3 次实际降息中能够提前发出 2 次预警信号，而基线策略永远无法捕捉政策转变。

### 2.6 三个准确率的解读（训练 82.7% > 最终 91.7% > 历史测试 66.7%？）

三个数字乍看矛盾，实际上各有成因：

**历史测试集最低（66.7%，42 次）** 是最可信的泛化估计。它通过分层随机抽样覆盖 2000-2024 年所有经济周期——零利率时代、金融危机、COVID 冲击——一组固定权重很难同时适应如此多样的环境，尤其加息（HIKE）召回率只有 37.5%，暴露了模型在非典型时期的局限。

**最终测试集偏高（91.7%，12 次）** 有三点结构性原因：① 12 次会议中 9 次是 HOLD（占 75%），类别高度集中，基线本身就是 75%；② 2025-2026 年与训练数据中最近的 2021-2024 年市场特征最相近，信号更清晰；③ 样本仅 12 个，统计置信区间极宽（约 ±15pp），单次运气成分不可忽视。

**结论**：历史测试集的 66.7% 是更保守、更诚实的跨周期泛化能力估计；最终测试集的 91.7% 反映近期特定市场环境下的实战表现。模型真实能力大概在两者之间，且对近期降息周期的适应性优于历史极端时期。

## 三、V6：从量化融合到多智能体推理

### 3.1 为什么要从 V5 迭代到 V6？

V5 已经是方法论上最完整的量化版本，但它本质上是一台**加权计算器**：

```
fusion_score = 0.40×语义 + 0.35×VAR + 0.10×市场 + 0.15×政治
```

这台计算器有两个无法解决的局限：

1. **黑箱输出**：只输出一个数字和一个标签（如"HOLD"），无法说明"为什么"——哪路信号起了决定性作用，各路信号之间是否存在矛盾，置信度从何而来。
2. **僵化权重**：四路信号的相对重要性在不同宏观背景下应该动态变化，但网格搜索出的固定权重无法自适应。比如 CPI 飙升时 VAR 信号理应权重更高，收益率曲线倒挂时市场信号更关键——固定权重无法捕捉这种情境依赖。

V6 的设计目标：**在保留 V5 全部量化信号的前提下，用 AI 推理替代固定权重，并让每次预测附带可读的推理链和风险归因**。

### 3.2 V6 架构设计

V6 是本系统唯一实现"真正并行多智能体"的批量预测版本（V1 是串行流水线，V2-V5 是函数管道）。

**五个 Agent 的分工**

| Agent | 角色 | 输入信号 | 任务 |
|-------|------|---------|------|
| A — 语义分析师 | 解读声明措辞 | LLM 鹰鸽分（滞后一期） | 判断声明倾向 + 置信度 |
| B — 计量经济学家 | 解读数量模型 | VAR 信号 + 宏观三指标 | 判断计量方向 + 关键变量 |
| C — 固收市场分析师 | 解读债市预期 | 收益率曲线百分位 | 判断市场隐含方向 |
| D — 政策环境分析师 | 解读外部压力 | EPU/VIX 百分位 | 判断政治压力影响方向 |
| E — 裁决 Agent | 综合四路报告 | A/B/C/D 全部输出 + V5 参考融合分 | 最终预测 + 风险归因 |

**关键设计：并行 + 汇聚**

```python
# 四个信号 Task 同时发出 LLM 请求
t_sem = Task(..., async_execution=True)   # Agent A
t_var = Task(..., async_execution=True)   # Agent B
t_mkt = Task(..., async_execution=True)   # Agent C
t_pol = Task(..., async_execution=True)   # Agent D

# 裁决 Task 自动等待四路完成后启动
t_arb = Task(..., context=[t_sem, t_var, t_mkt, t_pol])  # Agent E
```

`async_execution=True` 让 CrewAI 同时发出四个 DeepSeek 请求，而不是依次等待。裁决 Agent 通过 `context` 参数读取四路输出，不需要任何额外粘合代码。

**所有文字内容（角色、目标、提示词模板）全部配置在 `config.yml` 的 `v6_agents` 和 `v6_task_templates` 节点**，代码文件只含逻辑，不含字符串。

### 3.4 V6 的数据泄露与污染问题分析

这是 V6 独有、V2-V5 不存在（或程度更轻）的重要问题，必须诚实说明。

**（1）量化特征层——无泄露**

V6 的四路输入信号完全来自 V5 的特征矩阵，V5 已做严格的"走一步预测"设计：

- LLM 语义分：使用上次声明（shift(1)），不使用当次声明
- VAR 信号：截断到会议前，不含会议当天数据
- 市场 / 政治信号：历史滚动百分位，不含前视信息

**（2）LLM 推理层——存在知识污染风险**

这是 V6 独有的问题。裁决 Agent 和四个信号 Agent 都调用 DeepSeek-Chat，而 DeepSeek 的训练数据截止日期可能覆盖 2025 年部分时间段。

这意味着：当 Agent 看到"会议日期：2025-05-07"时，DeepSeek 内部可能已经"记得"那次会议美联储维持了利率，而不是通过分析输入的量化信号推理得出 HOLD 结论。

这种情况被称为**训练数据污染（Training Data Contamination）**，是所有使用预训练 LLM 进行历史回测时共同面临的挑战。

**（3）唯一可信的"干净"预测**

12 次测试中，只有最后一次——**2026-06-17 沃什首次主持**——大概率超出了 DeepSeek 的训练截止日，是真正意义上的未见数据。裁决 Agent 给出 HOLD，与实际一致，且推理过程清晰（鹰派立场 + 四路信号均衡）。

**（4）诚实结论**

| 层面 | 结论 |
|------|------|
| 量化特征（四路数值） | **无前视偏差**，V5 框架已处理 |
| LLM Agent 推理 | **存在知识污染风险**，2025 年历史数据可能在训练集内 |
| 91.7% 准确率 | **部分可信**：沃什那次是最干净的真实预测；其余 11 次存在 LLM 知识混入的可能 |
| 解决方案 | 持续运营——每次新 FOMC 会议才是真正的 out-of-sample 检验 |

### 4.2 箭头①：V1 → V2（从实时到系统，从定性到全量化）

**V1 的局限**：V1 是一次性实时预测系统，每次运行只分析当前一次会议，没有历史验证，无法量化模型的真实预测能力。权重全靠人工设定，缺乏数据驱动的依据。

**V2 做了什么**：

| 改进项 | 具体内容 |
|------------------------------------|------------------------------------|
| 历史批量分析 | 从美联储官网自动抓取 2000-至今 217 次声明，全部过 LLM 打分，存入数据库 |
| 四路全量化 | LLM 分、VAR 信号、收益率曲线市场信号、EPU/VIX 政治压力信号全部数值化 |
| 网格搜索 | 遍历 6012 个合法权重/阈值组合，在训练集上以准确率最大化为目标自动寻优 |
| 三表数据库 | `fomc_history`（会议记录）+ `semantic_history`（LLM语义）+ `political_quant_monthly`（月频宏观） |
| 走一步 VAR | 每次预测只用会议前 2 天截止的宏观数据，严格无前视偏差 |

**背后原理**：多信号融合的动机来自**没有任何单一信号全能**的事实。LLM 能读文字但不了解历史统计规律；VAR 懂数字但看不懂措辞变化；市场利率反映集体预期但有时滞后；政治压力捕捉尾部风险但信噪比低。四路融合，取长补短。

### 4.3 箭头②：V2 → V3（修复时间固定效应）

**V2 的问题：时间固定效应（Time Fixed Effects）**

V2 将 2000-2020 作为训练集，2021-2024 作为测试集。这带来一个系统性偏差：

```         
V2 测试集（2021-2024）全部处于疫后高通胀 + 快速加息周期
  → 该时期 HIKE 样本多，HOLD 基线仅 56.2%
  → 测试集"相对容易"，超越基线幅度（+9.4pp）被虚高
  → 模型可能只学到了"加息期的判断逻辑"，而非普遍规律
```

这个现象在计量经济学中称为**时间固定效应**：相邻年份的会议因共享同一宏观背景（如 2022-2023 的高通胀共识），在统计上并非独立同分布。

**V3 的修复：分层随机 80/20 抽样**

```         
全部 206 次历史会议（2000-2024）
     ↓
按 HOLD / HIKE / CUT 三类分别独立随机打乱（seed=42）
     ↓
  各类前 80% → 训练集（165次）
  各类后 20% → 测试集（41次）
     ↓
训练集和测试集各自均匀覆盖所有经济周期
（互联网泡沫、金融危机、零利率时代、加息周期……）
```

逻辑依据来自**随机对照实验（RCT）**的精神：通过随机分配，使两组在所有可观测和不可观测特征上分布一致，从而排除"所处时代"这一混淆变量。

**代价**：V3 随机打乱后，相邻两行可能对应 2003 年和 2018 年的会议，因此无法直接做"上一次会议"的滞后操作——这个限制在 V5 中得到解决。

------------------------------------------------------------------------

### 4.4 箭头③：V2 → V4（修复同期信息泄露）

**V2/V3 存在的根本性问题：同期信息泄露（Contemporaneous Leakage）**

V2 和 V3 均使用**当次 FOMC 声明**的鹰鸽分来预测**当次会议**的利率决策。这在逻辑上是有**巨大缺陷**的：

```         
关键事实：FOMC 声明原文与利率决策在同一时刻（美东时间下午 2:00）对外公布

也就是说，声明文本和决策结果是同一决策过程的两个输出，
用当次声明来"预测"当次决策，在现实中根本不可操作——
因为在决策公布之前，你根本拿不到声明原文。
```

这相当于用一个人出门后的穿搭（声明）来"预测"他出门前就已经决定的行程（决策）。

**V4 的修复：LLM 信号滞后一期（shift(1)）**

```         
V2/V3 逻辑（存在泄露）：
  第 N 次声明（鹰鸽分）→ 预测第 N 次决策     ✗

V4 修复后：
  第 N-1 次声明（鹰鸽分）→ 预测第 N 次决策   ✓ 真正前瞻
```

代码层面只有三行改动：

``` python
for col in ["hawkish_score", "inflation_concern", "growth_concern"]:
    df[col] = df[col].shift(1)   # 整体后移一期
df = df.dropna(subset=["hawkish_score"])  # 丢弃第一行（无前驱）
```

**经济直觉**：FOMC 政策具有路径依赖（forward guidance）。上次声明的措辞变化是当前政策立场的"领先指标"——鸽派措辞的积累，往往预告了下次行动方向。

**效果**：V4 在最终测试集（12次）准确率从 75% 跃升至 91.7%，其中最典型的修正是：

> 2026-06-17 沃什首次会议——V2/V3 读取沃什本人鹰派声明（+0.15）后误判 HIKE；V4 使用上次鲍威尔的鸽派声明（-0.25），融合分 -0.012，正确预测 HOLD。

**代价**：V4 沿用 V2 的时间切割，历史测试集集中在 2021-2024 加息周期，测试准确率（87.5%）有所虚高。

------------------------------------------------------------------------

### 4.5 箭头④：V3 + V4 → V5（两项修复同时生效）

**V5 要解决的问题**：根据控制变量的思思想， V3 和 V4 各解决了一个问题，且这两个问题都独立地表明能够有助于提高模型的预测精度。但能不能同时解决这两个问题？

障碍在于：两步操作**顺序不能颠倒**。

```         
❌ 错误顺序（先随机打乱再滞后）：
  随机打乱后：2003年 | 2018年 | 2007年 | ...
  shift(1) 后："上一次" = 毫无关联的另一年会议，失去经济含义

✓ 正确顺序（先滞后再打乱）：
  Step 1：按时间顺序计算 shift(1) —— 每次会议的 lag 特征锁定为真实的时间前驱
  Step 2：做分层随机 80/20 划分 —— 只影响哪些行进训练/测试，不改变 lag 值
```

**关键洞察**：滞后特征是**特征工程步骤**，必须在划分之前在时间顺序上完成。一旦 lag 特征被锁定在每条记录里，后续的随机划分不会破坏它的经济含义。

V5 的历史测试集（42次，随机 20%，跨 2000-2024）准确率为 66.7%（基线 64.3%，+2.4pp），这是两项修复都生效后更诚实的泛化评估——V4 历史测试集 87.5% 中有相当部分来自时代偏差，真实泛化优势约在 +2-3pp 量级。

------------------------------------------------------------------------

### 4.6 箭头⑤：V5 → V6（从量化融合到多智能体推理）

**V5 已经解决的问题**：时间固定效应、同期信息泄露、权重自动优化——方法论层面相当完整。

**V5 无法解决的问题**：

- 输出是黑箱（一个数字，一个标签），无法解释"为什么"
- 固定权重无法根据宏观背景动态调整
- 不符合笔试要求的"多智能体编排"真实架构

**V6 的设计思路**：不抛弃 V5，而是将 V5 的四路量化信号作为输入，分别交给四个专业 AI 分析师"解读"，再由裁决分析师综合四份报告做最终判断。

```
V5 做法（黑箱公式）：
  4个数字 → 加权求和 → 一个标签

V6 做法（可解释推理）：
  4个数字 → 4个AI分析师各自分析 → 4份自然语言报告
                                  → 裁决AI综合 → 预测 + 风险归因
```

**V6 与 V5 的准确率为何相同（91.7%）？**

V6 的裁决 Agent 看到了 V5 的融合分和参考预测，相当于有了量化锚点。Agent 的自然语言推理在大多数情况下会与量化锚点保持一致，只在边缘案例（信号矛盾时）才会出现分歧。这解释了两者准确率几乎相同的现象。

V6 真正的价值不在于提高准确率，而在于**将"为什么"这个问题答得有理有据**——每次预测都带有"主要驱动信号是哪路，各路信号是否矛盾，置信度高低"的完整分析。

## 六、现有不足与后续改进方向

### 6.1 同一漏判的根源：信号还不够早

三次降息中唯一漏判的是 **2025-10-29**，融合分 -0.288，距阈值 -0.40 差 0.112。LLM 分已经达到 -0.55（非常鸽），但 VAR 和市场信号仍显示 HOLD，拉高了融合分。

本质原因：VAR 是基于 FEDFUNDS / CPI / UNRATE 的 72 个月滚动窗口，反映的是宏观数据的历史趋势，对突然的"转向意愿"不够敏感。可能的改进方向： - 引入**Δhawkish**（鹰鸽分变化量）作为额外特征，捕捉措辞趋势加速 - 以 **Macro F1** 而非 Accuracy 为优化目标，迫使模型更重视少数类 CUT

### 6.2 新主席上任期建模不足

沃什 2026-05-22 上任，目前仅有 1 次会议记录（2026-06-17），样本严重不足。V1 的"主席行为蒸馏"智能体因数据稀少而效果有限。

改进方向：对 12 位 FOMC 投票委员分别建立鹰鸽立场画像，通过委员分歧程度（SEP 点阵图离散度）预测集体决策的不确定性。

### 6.3 样本量与评估稳健性

12 次最终测试集中每次预测正确与否影响准确率 8pp，统计置信区间偏宽。理想状态是持续运营——每次 FOMC 会议后自动并入数据库，最终测试集随时间自然增长，评估结果趋于稳定。

### 6.4 宏观数据时效性

FRED 数据存在 1-2 个月滞后（CPI 为上月数据）。可引入高频替代数据（商品期货、信用卡消费、Google 搜索指数）和实时新闻情绪指数，替换发布滞后的月度指标。

### 6.5 寻找工具变量以解决内生性问题

在现有的框架下，美联储表态、经济数据和市场信号等变量间不可避免地仍然存在内生性问题。接下来可以考虑巧妙地引入工具变量来解决模型的内生性问题。

### 6.6 V6 LLM 知识污染的根本性解决方案

如第三章 3.4 节所述，V6 Agent 推理层存在 DeepSeek 训练数据知识污染风险。根本性的解决方案是：

1. **持续运营**：每次新 FOMC 会议（约 8 周一次）才是真正的 out-of-sample 检验，LLM 不可能"记住"未来的结果。
2. **滚动评估窗口**：随时间积累新的真实预测记录，最终测试集自然增大，统计置信区间收窄。
3. **使用知识截止明确的模型**：在部署真实预测时选择知识截止日早于预测目标日的模型版本，确保 LLM 不含目标事件的训练信息。
