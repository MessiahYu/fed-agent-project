> **作者**：余青锋

## 一、项目概述

### 1.1 项目目标

本项目针对笔试题要求，搭建一个**基于多智能体编排的持续性经济事件预测系统**，预测美联储（Fed）年内议息会议的政策走向（加息 / 降息 / 维持不变），并对新任主席沃什（Kevin Warsh）的政策脉络进行推理，给出下次会议的决策方向预测。

### 1.2 项目要求完成情况

| 项目要求 | 完成情况 |
|---------|---------|
| 数据自动化感知智能体 | ✅ 自动抓取美联储官网声明（2000-至今）+ FRED API 实时宏观数据 |
| 语义特征提取智能体 | ✅ DeepSeek LLM 批量分析 217 次声明，输出可量化鹰鸽分（hawkish_score）|
| 决策与推演智能体 | ✅ VAR 计量模型 + 四路信号融合 + 网格搜索权重优化 |
| 数据触发→滚动预测→风险归因闭环 | ✅ 每次新会议自动触发数据更新、语义分析、信号融合、预测输出 |
| 内生性 / 因果识别论证 | ✅ 时间切割 + 走一步 VAR 严格防前视偏差，信号独立构建避免联立内生性 |
| 证据链可解释性 | ✅ 每次预测输出 key_evidence（关键证据条目）+ 信号拆解明细 |
| 沃什政策脉络推理 | ✅ 2026-06-17 首次会议分析（声明措辞偏鹰，实际维持不变）|

### 1.3 多智能体框架：CrewAI

本系统基于 **CrewAI** 框架搭建多智能体编排，而非 LangChain 或 AutoGen。

选择 CrewAI 的理由：
- 原生支持 **Role-Goal-Backstory** 角色设定，使每个智能体有明确分工和行为约束
- 内置任务依赖链（Task → Crew），天然适合"数据→分析→决策"的流水线结构
- 支持结构化输出（output_pydantic），保证信号数值可直接参与下游计算

```
CrewAI 编排结构

  LLM（DeepSeek）
       ↓
  Agent（角色 / 目标 / 背景）
       ↓
  Task（结构化输出：Pydantic 模型）
       ↓
  Crew.kickoff()
       ↓
  可量化特征 → 融合预测
```

### 1.4 机器学习组件分类

| 模块 | ML 类别 | 具体方法 |
|------|---------|---------|
| **权重优化（网格搜索）** | **有监督学习** | 以历史 FOMC 决策（HOLD/HIKE/CUT）为标签，在训练集上穷举 6012 个参数组合，最大化分类准确率 |
| **VAR 走一步预测** | **有监督学习**（时序）| 以历史利率序列为监督信号，向量自回归预测下期联邦基金利率方向 |
| **LLM 语义分析** | **零样本推理**（介于监督与无监督之间）| DeepSeek 基于预训练知识，无需额外标注样本即可对 FOMC 声明进行鹰鸽打分；本质是迁移学习 |
| **政治压力信号** | **无监督构建** | EPU / VIX 滚动百分位计算，不使用任何决策标签，纯特征工程 |

### 1.5 精度与时效性优势

**精度优势（vs 传统 VAR/VECM）**

传统计量模型只能处理数字，无法读懂声明措辞。本系统将 LLM 语义分析作为最高权重信号，捕捉"通胀忧虑措辞加重""前瞻指引措辞软化"等纯文本信息。在 2025-2026 年最终测试集上，系统在 3 次实际降息事件中预测正确 2 次，而"永远猜维持不变"的基线策略一次都抓不到。

**时效性优势**

每次 FOMC 会议声明发布后，系统自动触发抓取→语义分析→信号更新→预测输出的完整链路，全程无需人工干预，从声明发布到预测结果输出约 30-60 秒。

## 二、V3 模型：最终版本详解

### 2.1 基本原理

V3 的核心思想是将美联储的利率决策分解为**四路可量化信号**，通过数据驱动的权重融合，实现比单一模型更精准、更可解释的预测。

```
四路信号融合公式：

fusion_score = 0.35 × LLM语义分   （声明措辞传递的政策意图）
             + 0.35 × VAR计量信号  （宏观时序的历史规律）
             + 0.10 × 市场信号     （收益率曲线对未来利率的集体预期）
             + 0.20 × 政治压力信号  （不确定性/恐慌对政策空间的压缩）

决策规则：
  fusion_score > +0.20  →  预测 HIKE（加息）
  fusion_score < -0.35  →  预测 CUT（降息）
  其余                  →  预测 HOLD（维持不变）
```

各信号的数据来源与计算方法：

| 信号 | 数据来源 | 计算逻辑 |
|------|---------|---------|
| LLM 语义分 | 美联储官网 FOMC 声明原文 | DeepSeek 输出 -1.0（极鸽）到 +1.0（极鹰）|
| VAR 计量信号 | FRED：FEDFUNDS / CPI / UNRATE | 走一步 VAR，截断到会议前 2 天，无前视 |
| 市场信号 | FRED：DGS2 / DGS10 | 收益率曲线斜率（10Y-2Y）→ hike / hold / cut |
| 政治压力 | FRED：EPU 指数 / VIX | 滚动 36 月百分位加权，映射到 [-1, +1] |

### 2.2 设计思路：为什么这么做

**问题一：为什么要把 LLM 和 VAR 结合，而不是用其中一个？**

- 纯 VAR：看不懂文字，无法捕捉"声明措辞变化"这类前瞻性信号
- 纯 LLM：没有对历史周期统计规律的量化校准，可能被措辞迷惑

两者结合，互补短板。

**问题二：为什么 V3 用分层随机划分，而不是时间切割？**

时间切割（V2 做法）会引入**时间固定效应**：如果验证集集中在加息周期，而训练集都是平稳期，模型优化出的参数只是"适应了某一时代"，而非普遍规律。V3 将 2000-2024 年全部会议随机打散，使训练集和测试集都覆盖各种经济周期，得到更无偏的泛化评估。

**问题三：权重为什么是这个比例（0.35 / 0.35 / 0.10 / 0.20），而不是人工设定？**

权重由网格搜索在训练集（165 次会议）上自动优化，遍历 6012 个合法参数组合，以准确率最大化为目标。数据告诉我们 LLM 和 VAR 同等重要（各 0.35），而非人工拍脑袋。

### 2.3 V3 的核心成果

**成果一：历史回测**

在 206 次历史会议（2000-2024）的随机 20% 测试集（41 次）上：
- 准确率 65.9%，HIKE 召回率 100%（8/8 全部识别）

**成果二：最终测试集（2025-01 至 2026-06，12 次）**

| 会议日期 | 主席 | 实际决策 | V3 预测 | 结果 |
|---------|------|---------|--------|------|
| 2025-01-29 | Powell | HOLD | CUT | ✗ |
| 2025-03-19 | Powell | HOLD | HOLD | ✓ |
| 2025-05-07 | Powell | HOLD | HOLD | ✓ |
| 2025-06-18 | Powell | HOLD | HOLD | ✓ |
| 2025-07-30 | Powell | HOLD | HOLD | ✓ |
| 2025-09-17 | Powell | HOLD | HOLD | ✓ |
| 2025-10-29 | Powell | CUT | HOLD | ✗ |
| 2025-12-10 | Powell | CUT | **CUT** | ✓ |

总准确率：**9/12（75%）**

### 2.4 V3 真正有价值的突破：政策转折点识别

**原始 75% 准确率存在误导性**——12 次会议中 9 次是 HOLD，永远猜 HOLD 的"零智能基线"同样得到 75%。真正衡量模型价值的，是它能否在**政策转变的关键节点**发出正确信号。

2025 年 1 月以来共发生 **3 次降息**：

| 降息事件 | V3 预测 | 基线预测（永远 HOLD）|
|---------|--------|-------------------|
| 2025-10-29 CUT | HOLD ✗ | HOLD ✗ |
| 2025-12-10 CUT | **CUT ✓** | HOLD ✗ |
| 2026-01-28 CUT | **CUT ✓** | HOLD ✗ |

**V3 在 3 次降息中成功预判 2 次（67%），基线一次都预测不到（0%）。**

这说明系统不是在猜"大概率维持不变"，而是能在宏观信号积累到临界点时，提前发出政策转向警报。对于债券投资和利率衍生品交易，这正是具有实际交易价值的信息。

## 三、V1 / V2 / V3 迭代过程

### 3.1 版本设计对比

| 维度 | V1 | V2 | V3 |
|------|----|----|-----|
| **系统类型** | 实时单次预测 | 历史回测 + 实时预测 | 历史回测 + 实时预测 |
| **智能体数量** | 5 个 | 2 个核心模块 | 2 个核心模块 |
| **信号来源** | 语义 + VAR + 外部压力（定性）| 语义 + VAR + 市场 + 政治（全量化）| 同 V2 |
| **权重来源** | 人工设定 | 网格搜索（时间切割数据）| 网格搜索（分层随机数据）|
| **历史样本** | 1 次（当前会议）| 206 次（2000-2024）| 206 次（2000-2024）|
| **数据划分** | 无（单次运行）| 时间切割 | **分层随机 80/20** |
| **时间固定效应** | 不涉及 | 存在 | **已消除** |
| **可解释性** | 高（证据链输出）| 中（信号权重固定）| 高（权重变化有经济含义）|

### 3.2 各版本最优参数对比

| 参数 | 含义 | V1 | V2 | V3 |
|------|------|----|----|-----|
| `w_llm` | LLM 语义信号权重 | 人工约 0.4 | 0.45 | 0.35 |
| `w_var` | VAR 计量信号权重 | 人工约 0.3 | 0.20 | **0.35** |
| `w_market` | 市场信号权重 | 人工约 0.15 | 0.15 | 0.10 |
| `w_pol` | 政治压力权重 | 人工约 0.15 | 0.20 | 0.20 |
| `hike_thr` | 加息判断门槛 | 人工约 +0.3 | +0.25 | +0.20 |
| `cut_thr` | 降息判断门槛 | 人工约 -0.3 | -0.35 | -0.35 |

> V2→V3 最显著变化：VAR 权重从 0.20 大幅升至 0.35，LLM 权重从 0.45 降至 0.35。原因：随机划分覆盖多种经济周期后，宏观计量信号的跨周期稳定性得到数据验证，而单一声明语气的权威性在多元场景下有所稀释。

## 四、现有不足与后续迭代方向

### 4.1 数据时效性与信号覆盖面有待扩展

当前系统的宏观数据来自 FRED，存在 1-2 个月的发布滞后（如 CPI 数据在月末发布，覆盖上月数据）。可引入的增量信号包括：

- **实时替代数据**：商品期货价格、零售卡片消费数据、Google 搜索趋势（通胀预期代理）
- **FOMC 内部分歧指标**：SEP 点阵图的离散程度、会议纪要投票记录
- **国际联动**：欧央行 / 日本央行政策信号（货币政策联动效应）
- **新闻情绪**：基于新闻 API 的实时舆情指数，替代发布滞后的 EPU 月度数据

### 4.2 政策转折点预测能力有待提升

这是当前模型最核心的短板。若对每次会议均预测"维持不变"，在 2025 年 1 月至今的 12 次会议中准确率同样能达到 **75%**（12 次中 9 次确实是 HOLD）——这说明原始准确率作为评估指标本身存在缺陷，模型的真正价值在于识别少数"转变"事件。

当前 CUT 召回率 V3 为 66.7%（3 次中猜对 2 次），仍有一次（2025-10-29）被遗漏，主要原因是该次融合分为 -0.213，距降息阈值 -0.35 差距较大，VAR 和市场信号均未给出降息支持。

可能的改进方向：
- 将优化目标从 **Accuracy** 改为 **Macro F1**，迫使模型更重视少数类（CUT / HIKE）
- 引入"连续鸽派信号积累"特征：若连续 N 次鹰鸽分下行，自动提升降息信号权重
- 设置 CUT/HIKE 专属的早期预警子模型，与主融合模型并行运行

### 4.3 对美联储决策机制的理解有待深化

当前模型主要依靠声明文本和宏观数据，对美联储内部的政治生态（委员分歧、主席风格转变）建模不足。未来可考虑：

- **扩展 FOMC 委员追踪智能体**：对 12 位投票委员分别建立鹰鸽立场画像，动态追踪每位委员的最新演讲，用票数分布预测委员会集体决策
- **主席风格蒸馏增强**：当前 V1 已有 Agent 4（主席行为蒸馏），但 Warsh 仅有 1 次会议记录，数据严重不足。随着 Warsh 会议积累，持续更新其行为模型
- **国会压力信号**：国会对 Fed 的公开批评频率、听证会内容，可量化为"政治干预风险指数"

### 4.4 最终测试集样本量不足，评估稳健性有限

当前最终测试集仅有 **12 次会议**（约 2 年数据），在统计上置信区间过宽（约 ±25%），单次预测正确与否对准确率的影响达 8 个百分点。这使得基于 12 次样本的指标结论缺乏统计显著性。

可能的改进方案：

| 方案 | 做法 | 优点 |
|------|------|------|
| 扩大最终测试集 | 将 final_test 的起始时间从 2025 年往前移，纳入更多历史会议 | 样本量大，结论更稳健 |
| 时序交叉验证 | 用多个滚动窗口（如每次向后移 2 年）生成多组 train/test，取平均 | 兼顾时序性与样本量 |
| 重新调整比例 | 历史集 / 验证集 / 测试集按 60% / 20% / 20% 三段设计 | 验证集和测试集各自有足够样本 |

最理想的状态是**持续运营**：每次新 FOMC 会议结束后自动将结果并入数据库，最终测试集随时间自然增长，评估结果会随样本积累而趋于稳定。
