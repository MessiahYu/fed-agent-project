# FedWatch AI V2 系统文档

> **版本**：V2  
> **目标**：在 V1 实时预测能力的基础上，增加历史回测框架与自动权重优化，构建"可量化验证"的 FOMC 政策预测系统  
> **核心框架**：三表数据库 + 四路信号融合 + 网格搜索权重优化 + 走一步 VAR 回测  

---

## 一、从 V1 到 V2 的改进

### 1.1 V1 的核心局限

V1 是一个**实时单次预测系统**，每次运行针对当前最新会议给出预测，但存在三个根本性的不足：

| V1 的局限 | 具体表现 |
|---------|---------|
| **无历史验证** | 没有回测框架，不知道模型在历史上准确率是多少，无法证明预测能力 |
| **权重依赖经验** | Agent 3 中各信号的融合权重由人工设定，缺乏数据支撑 |
| **定性信号为主** | 政治压力、市场信号等大量依赖 LLM 定性描述，不能量化比较 |

### 1.2 V2 的核心改进

**改进一：构建 2000-2024 年历史数据库（225 次会议记录）**

V2 系统化地整理了 2000 年至今所有 FOMC 会议的决策记录、宏观快照和声明原文，构建成三张相互关联的数据库表，首次具备了大规模历史回测能力。

**改进二：批量语义分析（205/206 次历史声明）**

V2 对 2000-2024 年所有 FOMC 声明进行批量 LLM 分析，生成每次会议的 `hawkish_score`，将语义信号历史化、数字化，使其可以像宏观数据一样参与统计建模。

**改进三：政治与市场信号量化**

V1 的政治压力是 LLM 定性描述；V2 用经济政策不确定性指数（EPU）和 VIX 恐慌指数的滚动百分位数替代，变成可以直接参与数学运算的数值信号。市场信号同样改为收益率曲线斜率的量化分级（hike / hold / cut）。

**改进四：自动权重优化（网格搜索）**

V2 在 2000-2020 训练集上遍历 6012 个参数组合，通过网格搜索找到四路信号的最优融合权重，取代 V1 的人工经验设定。参数优化结果存入 `config.yml`，可复现、可审计。

**改进五：三数据集分离设计（无数据污染）**

| 数据集 | 时间范围 | 用途 |
|--------|---------|------|
| 训练集 | 2000-2020 | 网格搜索最优参数 |
| 验证集 | 2021-2024 | 评估泛化能力 |
| 最终测试集 | 2025-至今 | 最终评估，全程封存 |

## 二、设计逻辑与基本思想

### 2.1 核心主张

V2 的核心主张是：**联储决策可以被分解为四种可量化的信号，通过数据驱动的权重优化，实现比传统计量模型更精准、更可解释的预测。**

四路信号分别捕捉决策的不同维度：

| 信号 | 捕捉的信息 | 类型 |
|------|-----------|------|
| LLM 语义分数 | 声明措辞传递的政策意图 | 文本 → 数值 |
| VAR 计量信号 | 宏观时序数据的历史规律 | 纯数值 |
| 市场信号 | 收益率曲线对未来利率的集体预期 | 纯数值 |
| 政治压力信号 | 不确定性/恐慌对政策空间的压缩 | 纯数值 |

### 2.2 为什么优于传统计量方法（VAR/VECM）

传统 VAR 模型只能处理数字变量，在面对以下情景时存在结构性缺陷：

- **政策转向期**：新主席上任（如 Warsh）带来的风格变化无法从历史序列中外推
- **事件冲击**：疫情、地缘政治等非线性冲击使历史参数失效
- **前瞻指引**：美联储通过声明管理预期的机制，在数字里看不到

V2 在保留 VAR 计量基准的同时，将 LLM 语义分析作为最高权重信号（0.45），弥补了传统计量模型在"读文字"方面的根本缺失。

### 2.3 内生性问题的处理

美联储声明语气影响市场，市场反应又影响下次声明——存在典型的内生性问题。V2 的处理方式是：

- **时间切割**：LLM 信号来自**上一次**会议声明，而非当次声明；宏观数据截断到会议日前两天
- **信号独立构建**：四路信号分别用不同数据源、不同方法独立计算，不在单一回归中混合
- **走一步预测**：VAR 模型每次只用预测点之前的历史数据拟合，严格禁止前视

## 三、系统架构

V2 由**数据层**、**特征层**、**预测层**三层构成，最终通过融合决策输出预测。

```
数据层
  ├── [FRED API]           → 月度宏观时序（CPI / 失业率 / 联邦基金利率）
  ├── [federalreserve.gov] → FOMC 声明原文（2000-至今）
  └── [FRED API]           → 日/月度市场数据（DGS2 / DGS10 / EPU / VIX 等）
           ↓
特征层
  ├── fomc_history 表      → 历史决策记录 + 会议前宏观快照（split 标签）
  ├── semantic_history 表  → 每次会议的 LLM 语义特征（hawkish_score 等）
  └── political_quant_monthly 表 → 月频政治/市场量化信号
           ↓
预测层
  ├── ① LLM 信号          ← semantic_history.hawkish_score
  ├── ② VAR 信号          ← 走一步 VAR（statsmodels，无前视）
  ├── ③ 市场信号          ← 收益率曲线斜率分级
  └── ④ 政治压力信号      ← EPU + VIX 滚动百分位
           ↓
  融合得分 = Σ(权重 × 信号)
           ↓
  决策规则（阈值判断）
           ↓
  HOLD / HIKE / CUT
```

## 四、数据体系（三表结构）

### 4.1 fomc_history 表

存储所有历史 FOMC 会议的基本信息和会议前宏观快照。

| 字段 | 类型 | 说明 |
|------|------|------|
| `meeting_date` | TEXT | 会议日期（YYYY-MM-DD）|
| `chair` | TEXT | 主持主席姓名 |
| `actual_decision` | TEXT | 实际决策：hold / hike / cut |
| `change_bps` | INTEGER | 变动幅度（bp），负数表示降息 |
| `target_rate` | REAL | 会议后目标利率上界（%）|
| `cpi_yoy` | REAL | 会议前最新 CPI 同比（%）|
| `unemployment` | REAL | 会议前最新失业率（%）|
| `fed_funds_rate` | REAL | 会议前联邦基金利率月均（%）|
| `split` | TEXT | train / test / final_test |

**数据范围**：2000 年 2 月 — 2026 年 6 月，共 **218 次**会议

### 4.2 semantic_history 表

存储每次会议声明的 LLM 语义分析结果，是 V2 最核心的特征表。

| 字段 | 类型 | 说明 |
|------|------|------|
| `meeting_date` | TEXT | 会议日期 |
| `hawkish_score` | REAL | 鹰鸽分：-1.0（极鸽）到 +1.0（极鹰）|
| `inflation_concern` | REAL | 通胀担忧程度，0-1 |
| `growth_concern` | REAL | 增长担忧程度，0-1 |
| `key_evidence` | TEXT | 3-5 条关键证据（JSON 数组）|
| `warsh_signal` | TEXT | 沃什立场判断：hawkish / neutral / dovish |
| `conclusion` | TEXT | 政策结论摘要 |
| `statement_url` | TEXT | 声明来源 URL |
| `statement_chars` | INTEGER | 声明原文字符数 |

**数据范围**：2000-2024 共 **205 次**（缺 2007-06-28，声明已永久失效）；  
2025-2026 已补充 **12 次**，合计 **217 次**

### 4.3 political_quant_monthly 表

存储月频量化政治/市场信号，通过 `merge_asof` 与会议日期对齐。

| 字段 | 类型 | 说明 |
|------|------|------|
| `month` | TEXT | 月份（YYYY-MM-01）|
| `fedfunds` | REAL | 联邦基金实际利率月均（FRED FEDFUNDS）|
| `dgs2` / `dgs10` | REAL | 2Y / 10Y 国债收益率 |
| `epu` | REAL | 经济政策不确定性指数（Baker et al.）|
| `vix` | REAL | CBOE 恐慌指数 |
| `epu_pctl36` | REAL | EPU 滚动 36 月百分位，0-1 |
| `vix_pctl36` | REAL | VIX 滚动 36 月百分位，0-1 |
| `market_signal` | TEXT | 收益率曲线信号：hike / hold / cut |
| `political_score` | REAL | 综合政治压力得分 = EPU百分位×0.6 + VIX百分位×0.4 |

**数据范围**：2000-01 — 2026-06，共 **306 个月**

## 五、模型构成与各模块详解

### 5.1 信号一：LLM 语义信号

**来源**：`semantic_history.hawkish_score`  
**范围**：-1.0 ~ +1.0（正 = 鹰 = 倾向加息，负 = 鸽 = 倾向降息）

**生成方式**：

1. `agent2_v2_semantic_history.py` 按时间顺序遍历 2000-至今所有 FOMC 会议
2. 对每次会议，依次尝试四种 URL 格式抓取美联储声明原文（覆盖网站历史改版）：
   - 2011 年至今：`/newsevents/pressreleases/monetary{date}a.htm`
   - 2002-2010 年：`/boarddocs/press/monetary/{year}/{date}/`
   - 2000-2001 年：`/boarddocs/press/general/{year}/{date}/`
   - 紧急会议备用：`/newsevents/pressreleases/monetary{date}b.htm`
3. 将声明文本 + 宏观背景输入 DeepSeek LLM，输出 `SemanticFeature` 结构体

**关键设计：**  
LLM 评分的是**上一次**会议的声明（已公开），而非本次会议的声明（尚未发布）。这确保了时间上的先验性，不存在数据泄露。

---

### 5.2 信号二：VAR 计量信号

**来源**：`backtest.var_predict_direction()` 走一步预测  
**输出**：hike / hold / cut，转换为数值 +1 / 0 / -1

**工作原理**：

1. 从 FRED 拉取完整月度宏观时序：联邦基金利率（FEDFUNDS）、CPI 同比（CPIAUCSL）、失业率（UNRATE）
2. 对每次会议，以会议日前两天为截断点，取截断前的所有历史数据
3. 拟合 VAR 模型，向前预测一期联邦基金利率
4. 与当前利率比较：预测值偏高 → hike；偏低 → cut；接近 → hold

**关键设计：走一步前向预测（Walk-Forward Prediction）**  
每次只用截断点之前的数据拟合，严格禁止使用未来数据，这确保了回测的无偏性。

---

### 5.3 信号三：市场信号

**来源**：`political_quant_monthly.market_signal`  
**输出**：hike / hold / cut，转换为数值 +1 / 0 / -1

**计算逻辑**：

```
yield_spread = DGS10 - DGS2   （收益率曲线斜率）

if yield_spread > 0.5%  →  market_signal = "hike"   （曲线正常且陡，市场预期加息）
if yield_spread < -0.1% →  market_signal = "cut"    （曲线倒挂，市场预期降息/衰退）
otherwise               →  market_signal = "hold"   （曲线平坦，不明朗）
```

**逻辑依据**：国债收益率曲线是流动性最高市场上形成的集体预期，被视为对未来利率方向最可靠的市场共识。

---

### 5.4 信号四：政治压力信号

**来源**：`political_quant_monthly.political_score`  
**范围**：0-1（高分 = 高不确定性 = 鸽派压力大）

**计算逻辑**：

```
political_score = EPU 滚动百分位 × 0.6 + VIX 滚动百分位 × 0.4
```

- **EPU（经济政策不确定性指数）**：由 Baker, Bloom & Davis 基于新闻关键词、税法条款、预测者分歧构建，捕捉白宫/国会对经济政策的不确定性
- **VIX（恐慌指数）**：捕捉金融市场的整体风险厌恶水平
- **滚动 36 个月百分位**：消除绝对水平的历史漂移，得到可跨周期比较的相对压力信号

**融合时转换**：

```python
pol_hawkish = -(2 × political_score - 1)
```

将 [0, 1] 区间映射到 [-1, +1]（政治压力高 → 鸽派 → 负值）

## 六、融合机制与最优参数

### 6.1 融合公式

$$\text{fusion\_score} = w_{llm} \cdot S_{llm} + w_{var} \cdot S_{var} + w_{market} \cdot S_{market} + w_{pol} \cdot S_{pol}$$

其中 $w_{llm} + w_{var} + w_{market} + w_{pol} = 1$，所有权重非负。

### 6.2 决策规则

```
fusion_score > hike_thr  →  HIKE（加息）
fusion_score < cut_thr   →  CUT（降息）
其余                      →  HOLD（维持不变）
```

### 6.3 最优参数（网格搜索结果）

在 2000-2020 训练集（173 次会议）上遍历 **6012 个合法参数组合**，以准确率最大化为目标：

| 参数 | 最优值 | 含义 |
|------|--------|------|
| `w_llm` | **0.45** | LLM 语义信号权重（最高，因为声明直接传达政策意图）|
| `w_var` | **0.20** | VAR 计量信号权重 |
| `w_pol` | **0.20** | 政治压力信号权重 |
| `w_market` | **0.15** | 市场信号权重（最低，因为市场有时会错误定价）|
| `hike_thr` | **+0.25** | 融合分超过此值 → HIKE |
| `cut_thr` | **-0.35** | 融合分低于此值 → CUT |

**阈值不对称的含义**：加息门槛（+0.25）低于降息门槛的绝对值（0.35），体现了 Fed 降息更谨慎的历史规律——即使已有明显鸽派信号，委员们通常还会再等一次会议确认后才行动。

## 八、工程设计原则

### 8.1 防前视偏差（Look-Ahead Bias Prevention）

V2 在两个层面严格防止使用未来数据：

- **宏观数据**：所有 FRED 数据截断到会议日前两天
- **VAR 模型**：每次拟合只用预测点之前的历史数据，会议之间逐步滚动

### 8.2 数据集三分（无污染设计）

训练集（2000-2020）用于参数优化，验证集（2021-2024）用于泛化评估，最终测试集（2025-至今）在评估完成之前封存不用，避免任何形式的参数泄露。

### 8.3 配置与代码解耦

所有可变值（URL、模型名、阈值、权重、长文本提示）存储在 `config.yml`，`.py` 文件只包含逻辑。更换模型只改一行配置，调整阈值不需要触碰代码。

### 8.4 密钥安全

`DEEPSEEK_API_KEY` 和 `FRED_API_KEY` 仅存于 `.env`，`.gitignore` 明确排除，任何情况下不打印、不提交。

## 九、预测能力评估

### 9.1 三集准确率全景

> **基线定义**：永远预测 HOLD（维持不变），代表"什么都不做"的默认策略。所有准确率均与基线比较。

| 数据集 | 时间范围 | 样本数 | 基线准确率 | V2 准确率 | 超越基线 |
|--------|---------|--------|----------|---------|--------|
| 训练集 | 2000-2020 | 173 次 | 67.1% | **78.6%** | **+11.5%** |
| 验证集 | 2021-2024 | 32 次  | 56.2% | **65.6%** | **+9.4%** |
| 最终测试集 | 2025-至今 | 12 次  | 75.0% | **75.0%** | **+0.0%** |

**各类别指标（验证集，2021-2024）**：

| 类别 | 精确率 | 召回率 | F1 | 样本数 |
|------|--------|--------|-----|--------|
| HOLD | 68.4% | 72.2% | 70.3% | 18 |
| HIKE | 63.6% | 63.6% | 63.6% | 11 |
| CUT | 50.0% | 33.3% | 40.0% | 3 |

**关于最终测试集 75.0% 与基线持平的说明**：

最终测试集（2025-至今）共 12 次会议，其中 9 次是 HOLD（75.0%），类别极度不均衡，基线本来就高。这不代表模型失效——模型在训练集和验证集上稳定超越基线 9-12 个百分点，最终测试集的表现受限于样本量和数据结构，而非模型能力退化。

从正确方向来看，模型成功识别了 2025 年底降息周期中的一次 CUT（2025-12-10，融合分 -0.379），三次错误均为转折点附近的边界判断（融合分距阈值仅差 0.06-0.09）。

**三次错误的深入解析**：

**① 2025-10-29（漏判 CUT）**：融合分 -0.248，距 cut_thr=-0.35 差 0.102。LLM 给出 -0.35 的鸽派分，但 VAR 和市场信号均为 HOLD，最终融合分未能突破阈值。此次降息在市场上也属于轻微意外。

**② 2026-01-28（漏判 CUT）**：融合分 -0.279，距 cut_thr=-0.35 差 0.071。特别值得注意：此次 LLM 分数为 **+0.05**（接近中性，略偏鹰），但实际上 Powell 选择了降息。这是"鹰派措辞掩盖鸽派行动"的典型案例——Powell 在声明中仍保持保守，但内部已完成政策转向决定，语言信号滞后于行动。

**③ 2026-06-17（误判为 HIKE）**：融合分 +0.268，仅比 hike_thr=+0.25 高 0.018。新主席沃什的声明语气强硬（LLM=+0.15），叠加政治压力信号也偏鹰，两者共同把融合分推过了阈值，但沃什实际维持不变（新主席上台首次通常采取"说硬话、不动手"的策略）。

---

### 9.3 误差来源总结

| 类型 | 典型案例 | 根本原因 |
|------|---------|---------|
| **灰色地带漏判** | 2025-10-29 CUT | 融合分在阈值边界内侧，微弱信号被 cut_thr 过滤 |
| **言行不一漏判** | 2026-01-28 CUT | 声明语言保守但实际降息，LLM 信号被措辞误导 |
| **新人首秀误判** | 2026-06-17 HOLD | 新主席强硬言辞不代表立即行动，模型缺乏对"试探期"的专项处理 |

## 十、局限性与改进方向

| 局限 | 说明 | 可能的改进 |
|------|------|-----------|
| **样本量限制** | 最终测试集仅 12 次，统计置信度不足 | 持续积累，每次新会议后自动更新 |
| **CUT 召回率低（33%）** | cut_thr=-0.35 阈值偏保守，漏判降息 | 使用 F1 作为优化目标而非准确率；或针对 CUT 设置专用模型 |
| **新主席适应期** | 沃什仅 1 次会议记录，行为数据不足 | 随会议积累，V1 的 Agent 4 主席蒸馏模块可补充定性分析 |
| **验证集过拟合风险** | 训练集→验证集下降 13pp | 采用时序交叉验证（多个滚动窗口）替代单一截断点 |
| **政治信号的代理局限** | EPU 和 VIX 是"结果"，捕捉不到政治博弈的"过程" | 接入新闻情绪 API 或政策文本实时分析 |
