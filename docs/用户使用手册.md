## 一、项目是什么（60 秒速览）

本项目预测**美联储 FOMC 议息会议的利率决策方向**（加息 / 降息 / 维持不变）。

核心做法是将四路信号融合： 1. **LLM 语义分**：用 DeepSeek 分析每次 FOMC 声明的鹰鸽程度（-1.0 极鸽 → +1.0 极鹰） 2. **VAR 计量信号**：用向量自回归模型，基于利率/通胀/失业率历史数据预测方向 3. **市场信号**：收益率曲线斜率（10Y-2Y） 4. **政治压力信号**：EPU 指数 + VIX 的滚动百分位

系统从 2000 年以来的 217 次 FOMC 会议中学习，在 2025-2026 年的 12 次最新会议上验证。最终版本（V6）在这 12 次中预测正确 11 次（**91.7%**）。

## 二、六个版本一句话总结

| 版本 | 一句话 | 最终测试集准确率 |
|-----------------|------------------|-------------------------------------|
| V1 | CrewAI 5智能体实时单次预测（含主席蒸馏、影响网络分析） | 无批量统计 |
| V2 | 引入历史批量回测 + 网格搜索权重优化 | 75%（9/12） |
| V3 | 修复时间固定效应（分层随机 80/20 划分） | 75%（9/12） |
| V4 | 修复同期信息泄露（LLM 信号滞后一期） | **91.7%（11/12）** |
| V5 | V3 + V4 同时修复（方法论最完整版） | **91.7%（11/12）** |
| **V6** | **在 V5 量化信号基础上叠加 4 并行信号 Agent + 1 裁决 Agent，输出推理链和风险归因** | **91.7%（11/12）** |

### V6：并行多 Agent 推理（需要 DEEPSEEK_API_KEY + FRED）

V6 以 V5 的特征矩阵为输入，将四路量化信号交给四个并行 AI 分析师，再由裁决 Agent 综合输出最终预测和风险归因。

**架构说明**：

-   Agent A（语义分析师）：分析 LLM 鹰鸽评分反映的声明倾向
-   Agent B（计量经济学家）：结合宏观背景解读 VAR 模型信号
-   Agent C（固收市场分析师）：解读收益率曲线的市场隐含预期
-   Agent D（政策环境分析师）：判断 EPU/VIX 外部压力对决策的影响
-   Agent E（裁决 Agent）：等待四路报告，按权重综合，给出最终判断和风险归因

``` bash
# 运行单次（最近会议，约 30-60 秒）
python v6/pipeline_v6.py

# 运行全部 12 次（约 5-10 分钟，60 次 LLM 请求）
python v6/pipeline_v6.py --all
```

**注意**：V6 每次预测发出 5 个 LLM 请求（4 并行 + 1 串行裁决），全量运行 12 次共 60 个请求，需要稳定的网络和足够的 API 额度。

> **数据污染说明**：V6 的量化特征层（四路数值）无前视偏差；但 DeepSeek 训练数据可能包含 2025 年部分 FOMC 会议信息，LLM 推理层存在知识污染风险。2026-06-17 沃什首次会议是最干净的 out-of-sample 预测。

## 十一、常见问题

**Q：运行时报 `ModuleNotFoundError`？**\
A：确认已激活 `fed-agent` 环境（`conda activate fed-agent`），并在项目根目录运行脚本。

**Q：FRED 数据拉取超时？**\
A：FRED API 偶尔限速，等 30 秒后重试即可。也可在 `config.yml` 的 `backtest.var_window` 中缩小滚动窗口。

**Q：V3 划分和 V4/V5 划分冲突？**\
A：V4/V5 在代码内**显式重建**自己的划分（不依赖数据库 `split` 列），与 V3 不冲突。运行顺序不影响结果。

**Q：想修改参数搜索空间？**\
A：修改 `v2/weight_optimizer.py`（或 `v4/`、`v5/` 各自对应文件）顶部的 `W_GRID`、`HIKE_GRID`、`CUT_GRID` 列表。

**Q：如何查看 V1 的主席蒸馏分析结果？**\
A：直接运行 `v1/pipeline.py`，或单独运行 `v1/agent4_chair_distill.py`（需要 DeepSeek API）。注意主席蒸馏在 V2-V5 中未作为量化特征使用，仅存在于 V1 实时分析流水线。

**Q：V6 的结果每次运行会不一样吗？**\
A：有可能。V6 的四路信号 Agent 和裁决 Agent 都调用 LLM，LLM 存在一定随机性（temperature \> 0）。但由于裁决 Agent 会参考 V5 的量化融合分作为锚点，在信号明确的情况下结果通常一致。如需确定性输出，可修改 `_llm()` 函数添加 `temperature=0` 参数。

**Q：V6 与 V1 有什么区别，都是多 Agent吗？**\
A：V1只是初期头脑风暴版本。V1 是串行流水线（五个 Agent 依次执行），分析的是当前最新一次会议（实时预测）；V6 是并行架构（四个 Agent 同时运行后汇聚），分析的是历史 12 次封存测试集，目的是验证批量预测能力和可解释性。
