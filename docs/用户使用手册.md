## 一、项目是什么（60 秒速览）

本项目预测**美联储 FOMC 议息会议的利率决策方向**（加息 / 降息 / 维持不变）。

核心做法是将四路信号融合：
1. **LLM 语义分**：用 DeepSeek 分析每次 FOMC 声明的鹰鸽程度（-1.0 极鸽 → +1.0 极鹰）
2. **VAR 计量信号**：用向量自回归模型，基于利率/通胀/失业率历史数据预测方向
3. **市场信号**：收益率曲线斜率（10Y-2Y）
4. **政治压力信号**：EPU 指数 + VIX 的滚动百分位

系统从 2000 年以来的 217 次 FOMC 会议中学习，在 2025-2026 年的 12 次最新会议上验证。最终版本（V5）在这 12 次中预测正确 11 次（**91.7%**）。

| V5 | V3 + V4 同时修复（方法论最完整版）| **91.7%（11/12）**|

## 五、环境配置

### 5.1 前置要求

| 要求 | 说明 |
|------|------|
| Anaconda / Miniconda | 推荐 Anaconda |

```bash

# 安装依赖
pip install crewai crewai-tools openai python-dotenv
pip install pandas numpy pyyaml requests beautifulsoup4
pip install statsmodels fredapi streamlit
```

### 5.3 配置 API 密钥

在项目根目录创建 `.env` 文件（不要提交到 git）：

```
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
FRED_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

> **最低配置**：只运行 V2-V5 的最终测试集评估，只需要 `FRED_API_KEY`。
> `DEEPSEEK_API_KEY` 仅在运行 V1 或重建语义数据库时需要。

## 十一、常见问题

**Q：运行时报 `ModuleNotFoundError`？**  
A：确认已激活 `fed-agent` 环境（`conda activate fed-agent`），并在项目根目录运行脚本。

**Q：FRED 数据拉取超时？**  
A：FRED API 偶尔限速，等 30 秒后重试即可。也可在 `config.yml` 的 `backtest.var_window` 中缩小滚动窗口。

**Q：V3 划分和 V4/V5 划分冲突？**  
A：V4/V5 在代码内**显式重建**自己的划分（不依赖数据库 `split` 列），与 V3 不冲突。运行顺序不影响结果。

**Q：想修改参数搜索空间？**  
A：修改 `v2/weight_optimizer.py`（或 `v4/`、`v5/` 各自对应文件）顶部的 `W_GRID`、`HIKE_GRID`、`CUT_GRID` 列表。

**Q：如何查看 V1 的主席蒸馏分析结果？**  
A：直接运行 `v1/pipeline.py`，或单独运行 `v1/agent4_chair_distill.py`（需要 DeepSeek API）。注意主席蒸馏在 V2-V5 中未作为量化特征使用，仅存在于 V1 实时分析流水线。
